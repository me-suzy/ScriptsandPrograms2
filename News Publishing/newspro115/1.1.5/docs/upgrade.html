Utopia News Pro Upgrade Documentation - 1.1.5

Utopia News Pro has been designed to make upgrading from version to version as quick and easy as possible. Just follow the instructions and you should not run into any problems. If you do, send an email to utopiasupport@gmail.com.

The first step is to unzip the new version .ZIP archive, ensuring to retain the directory/file structure.

Backing Up

The first step in the upgrade process is backing up. Always back up your MySQL database before attempting an upgrade. If you have made any changes to the default files from the previous version, make sure to back up these files too.

Uploading The Files

Upload all files from the /upload/ directory except config.inc.php and install.php ensuring to retain the directory/file structure.
From the directory /upgrade/, upload the files upgradeX.php, where X is the number script that you will need to use.

Use the following upgraders for each version:
From 1.0.0: upgrade1.php, upgrade2.php, upgrade3.php, upgrade4.php, upgrade5.php, upgrade6.php
From 1.1.0: upgrade2.php, upgrade3.php, upgrade4.php, upgrade5.php, upgrade6.php
From 1.1.1: upgrade3.php, upgrade4.php, upgrade5.php, upgrade6.php
From 1.1.2: upgrade4.php, upgrade5.php, upgrade6.php
From 1.1.3: upgrade5.php, upgrade6.php
From 1.1.4: upgrade6.php

Running The Upgrader(s)

Once all files have been uploaded to the server, open your web browser and navigate to http://www.yourdomain.com/unpdirectory/upgradeX.php, replacing www.yourdomain.com, unpdirectory, and upgradeX.php with the correct values for your site.
You must run each upgrade script progressively in the given order.
All steps of the upgrade are extremely straightforward. Once each step is completed, simply proceed to the next step. You will not be asked to submit any information.

Completion

At the end of each upgrade script, you will either be told to proceed to the next upgrade script (if it exists on the server) or to the administration panel (if the next upgrade script does not exist on the server).
Once you have run through all available upgrades available for your version of UNP, you will be sent to the administration panel. When this happens, immediately remove all upgradeX.php scripts from the server.

After Upgrading

After upgrading, immediately remove all upgradeX.php scripts from the server, as they can pose a security risk.

If you were previously running UNP 1.1.0 and you had customized your newsbit.php, you will need to carry over the changes to the new newsbit, since template handling has changed between versions. Copy the template contents of the newsbit.php file to the new news_newsbit in the template editor of the administrator control panel. Surround variables with { and }, instead of the old <?= and ?> (ie. {$var} instead of <?=$var ?>).
Note: These changes are unnecessary if you did not customize your newsbit.

If you plan on using the avatars feature introduced in version 1.1.2, you must ensure that the new /images/avatars directory is writable (CHMOD 777).

Once the upgrade scripts are removed, log into the administration panel, and navigate to the settings page. Configure any new settings that were introduced in the new version.
