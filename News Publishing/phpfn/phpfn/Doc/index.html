Introduction

PHPFreeNews is a free PHP script which allows you to display "news" articles on your own website. This allows you (or your friends or colleagues) to easily update new articles without having to naually edit the web page HTML. The script allows you to control the content and layout of the news articles, and allows full maintenance and audit of the news items. Features include:

  • Creation, Editing and Posting of news articles
  • Associate an image with each article
  • Allow users to provide comments on articles
  • Allows user to rate articles
  • Full back-office maintenance and control facilities.
  • Highly configurable
  • Frequent upgrades and new features
  • Free!

The latest version of the script can be downloaded from the PHPFreeNews website, and you should ensure that you are using the latest version before you install this script.

Requirements

PHPFreeNews has very few system requirements:

  • A webserver. The script has been developed on Apache 2.x running on Linux, but it should work with any PHP/MySql compatible server
  • PHP support. The script has been developed on PHP4 but it should work happily with other versions
  • MySql. The script has been developed on MySql 4 and requires MySql 4 or later to function.

Installation

1. Get hold of the script from the Downloads page at www.phpfreenews.co.uk. This is a ZIP archive with pathnames.

2. Extract the zip file contents (retaining the folder structure), and upload this to your website. The suggested location is www.yourservername.com/phpfn (or www.yourservername.com/News), and the rest of this page assumes that is where you have placed the script.

3. Edit the configuration file Config.php in the /phpfn/Config folder. In here you should specify your MySql details and also the name of your news database (which you must create yourself). The first section in the configuration file is mandatory; the remainder can be changed if you wish to alter the default behaviour.

4. Point your browser at www.yourservername.com/phpfn/Install.php (e.g. www.yourservername.com/phpfn/Install.php) and follow the prompts. This will now set up your database structure and will populate the tables with some very basic sample data. If you see any errors then you must correct these and then try again.

5. Edit the Config.php file again (the copy on the webserver) and set the "AllowInstall" value to false - this will prevent anybody else from re-running your installation script!!!

6. Err....there's no step 6, you've finished! Point your browser at www.yourservername.com/phpfn/Admin.php and login with user=Admin, password=Admin.

If you are upgrading an existing installation then please point your browser at the appropriate file within the Upgrade subfolder. This will upgrade your database structure, so be sure to take a backup of your data first! You must upgrade from the previously released database structure, e.g. work your way through the Upgrade scripts. However, it's much better to keep "on top" of upgrades, so that you only need to apply a single upgrade.

Once you have installed the software, either as a clean install or as an upgrade, then please delete the entire Upgrade subfolder.

Including News On Your Site

Once you've configured your script and setup your templates etc, you'll probably want to display the news on your site! This is actually very easy to do - though there's lots of ways to do it. The simplest option is to add the following line to your existing PHP page:

<?php include "phpfn/News.php" ?>

This will display the news on your site. If nothing shows, or if you see an error, then now would be a good time to resolve it. But everything should run smoothly.

HOWEVER...the chances are that you saw nothing on your site. Why? Well, because articles have to be assigned to categories, and you need to say which news-category you want to show. Therefore the following examples are better:

<?php $CatID='1'; include 'phpfn/News.php' ?>

<?php $CatID='1,2,3'; include 'phpfn/News.php' ?>

The exception to this is if you change the $ShowIfNoCat value in the Config file, as this will display articles even if they have been assigned no categories.

Note that the values set up in the Configuration file will take effect each time you display the news. If you want to override these on a per-page basis (e.g. display the news twice on one page, with each listing showing different news items) then you MUST include the file Override.php. So for example, let's say your default "news mode" in the config file says "$NewsMode = 2", but you want to change this to "$NewsMode = 1" for one page, then you would do this:

<?php include "phpfn/Override.php"; $NewsMode=1; include "phpfn/News.php" ?>

The same logic can be applied to override any of the config-file values, on an ad-hoc basis. Note that any values you override will remain overridden for all calls to News.php on that page. Therefore if you override a value for a one-off instance you might then have to override it back again, depending upon your requirements. If you don't include Override.php then your "extra values" will be ignored.

If you want to just display news headlines instead of news then you should use "phpfn/NewsHeadlines.php"instead of "phpfn/News.php"

Examples

Display simple news:
<?php include "phpfn/News.php" ?>

Display news, overriding the display-mode
<?php include "phpfn/Override.php"; $NewsMode=1; include "phpfn/News.php" ?>

Display news, displaying one specific article (Article IDs 17, 23 and 46)
<?php include "phpfn/Override.php"; $ArticleList='17, 23, 46'; include "phpfn/News.php" ?>

Display news, displaying archived-items as well as normal items:
<?php include "phpfn/Override.php"; include "phpfn/News.php" ?>

Display news, displaying only live posts, and only displaying one category:
<?php include "phpfn/Override.php"; $ExcludeArchivedPosts=1; $CatID='1'; include "phpfn/News.php" ?>

Display news, displaying only live posts, but also displaying Archived posts if a search is underway:
<?php include "phpfn/Override.php"; $ExcludeArchivedPosts=(isset($_POST['Match']) ? 0 : 1); include "phpfn/News.php" ?>

Display the search form above the news:
<?php include "phpfn/NewsSearchForm.php"; include "phpfn/NewsHeadlines.php" ?>

Display news, but allow the user to filter the category themselves:
<?php include "phpfn/NewsCategoryForm.php"; include "phpfn/News.php" ?>

Templates

Templates allow you to apply a fixed but flexibly design to your news article. You can set up as many templates as you like, and then associate a single template with each article.

Templates are designed using standard HTML tags, such as <TABLE>, <TR> etc. But in addition, there are certain "special words" you can use in these templates to show your news. For example, if your template consists of the following:

<TABLE><TR><TD>Posted on {newsdate} at {newstime} </TD></TR></TABLE>

Then your news will show the date and time the article was published.

Several tags are available, and they are shown within the Template Maintenance page.

Note that Templates are made up from three parts, each one controlling the display of Headlines, Short Posts and Long Posts. You can customise them all however you wish, and PHPFreeNews will apply the appropriate formatting according to the type of listing being displayed. For example, if you are displaying only news headlines then only the "Headline" element of the Template will be applied.

The exact template codes are as follows:

  • {headline} - the news headline;
  • {author} - the name of the person writing the news;
  • {newsdate} - the datestamp on the news article;
  • {newstime} - the timestamp on the news article;
  • {timesread} - the number of times the article has been read;
  • {image} - the image associated with the article, if any; There are three variants on this - {imagel}, {imagec} and (imager} and these will give your image Left, Centre or Right alignment respectively.
  • {id} - the internal ID of the article - useful for creating your own links;
  • {news} - the actual news (short or long);
  • {rating} - the number of votes, if you have enabled the voting system;
  • {comments} - a link to Comments functionality, if you have enabled it;
  • {readmore} - a link to the long-post, if there is one .

Most of these are optional, and will be replaced with blanks if applicable (e.g. the {image} tag will be replaced with blanks if an article has no associated image)

  User-Defined Codes

User-Defined Codes allow you to create your own "meta-tags" which will be used when news is displayed.

So for example, if you always write the words "Smith and Sons - Your Online Partner" in your articles you could setup a user-defined code called "{tagline}" and just use "{tagline} in your articles. When the article is published it will show your actual tagline instead.

This is a great way of maintaining consistency across your articles. Change the user-defined code, and all your articles are immediately updated!

Terminology

PHPFreeNews uses a few terms which you may or may not be familiar with. Here's brief explanation of them:

Archive: Push articles into the Archives so that they are removed from display. This can be done on a per-article basis or by the age of the article. Useful if you want to remove articles from display but still retain them for reference.

News Feed: An RSS feed, similar to the feeds appearing on lots of websites. This allows users to "subscribe" to your news articles.

Locked: A Locked article is one which cannot be changed. It cannot be edited, made Sticky, etc. Locked articles are also explicity excluded from the Archive and Purge routines. Only users with the correct privileges can lock and unlock articles.

Purge: Allow you to permanently delete articles from your database. This is a simply feature, useful only for having a spring-clean.

Sticky: An article which is to appear as though it is "stuck" at the top of the list. Useful for displaying long-running news articles which you don't want to get "bumped" down the list by more recent articles.

Template: A default layout for your new articles.

Visible: Only Visible articles will be displayed. This cannot be overridden.

Help and Support

PHPFreeNews is well-supported, both by the author and by a growing community of supporters. If you experience a problem or you need some assistance, your first port of call should be the Forums. Here you will find answers to many questions, or you can raise a new question. Please search for answers before posting a new question!

If you find that you are unable to login to the site, i.e. you forget the Admin password, then there's a Rewrite Password facility. Point your browser at http://website/InstallDir/RewritePassword.php. You will be prompted to enter a username - this is the MySql Username which you have in your Config file. If you supply the correct credentials then the master administrator (User ID 1) will have its password reset to the same as your Config-file password.

If you can't get a satisfactory solution from the forum (which is highly unlikely) then please get in touch with me via the website.
