Introduction:
What is leads4web?



Leads4web is a webbased CRM-Application.

CRM means Customer Relationship Management and stands for the management of customer related information of all kind.

With leads4web/3 you can manage your contact data within a team. Everyone within the team has certain rights and belongs to one or maybe more groups. So you can have a group "admin", a group "sales", whatever you want.

The admin decides about permissions and the general structure of the groups, depending on what suits your company.
Leads4web/3 is not a simple contact information management - it is a very flexible tool when it comes to organize information for more than one user.

Quick Tour:

Your first login:
This little tutorial supposes that you already installed leads4web. Information about installation is found in the directory "install". There you can find the login and password for the first access as well.

A short look:
With leads4web/3 you can manage and search your contacts, company information, receive email, write notes and create todos and so on.

These features are explained in detail in the following sections.

If you want to continue the quick tour, we'd like to show you how to change your password and create new users (leads4web/3 is a multi-user CRM System) first.

Changing your Password:
Open the configuration menu and click on "user info".

On the next page, enter your new password (twice, for verification) and click the button "save". We strongly recommend you to change your password, as everybody can easily find out about the default values.

The new password will be saved and you can check out if it is working by loging out and loging in again with your new password

Logging in as another user:
Click on "User management" in the Configuration menu and wait a second (or two ;) for the applet to be loaded. What you will see is something like the lefthand picture.

Moving your mouse over the default group "websphere" will result in a popup window having a link which lets you add a new user to that group.

Click on that link and fill out the form in the new window. If you don't want the user to become Admin of that group as well, uncheck the appropriate box.

Click on "the user has been added" to close the window and wait for the results in the main window: You should see the new user (and you can login now as the new user with the password you've just given).

We'll try that in the next step of this quick tour.

Creating new contact entries I:
Click the link "New Contact" in the tree menu. If you cannot find this link, click on "Contact" and the submenu will appear.

The "Search" and the "Extended search" items and all the other elements of the tree will be explained in the appropriate sections of "Functionality".



You can use the buttons shown above to controll the behavior of the tree. With these you can expand and collapse the submenus, save your preferred view, refresh the tree, get this help and log out.

Creating new contact entries II:




Create three new contacts by clicking on "new contact" in the tree menu. For the first contact, choose "websphere" as group and "read and write" as permission rights. For the second contact choose "websphere" and "readonly", the third one should be set as "private".

Remember that you are logged in as the new user you created some steps ago. What you get should look something like that:



The first contact is marked with a green icon, as it can be read and changed by anyone who belongs to the group "websphere", the default group. The third contact is marked with a red icon, as it is private (to you), whereas the second contact is green as well. It is marked green because you can read and change it - if is was a contact created by someone else - readonly - then there would be a yellow icon.

Logging in as admin again:
Log in as "admin" again (remember that you might have changed your password before) and click on "Contacts -> Search" in the tree menu.

Make sure you've choosen the right view (i.e. in the upper right the current group should be "websphere" or "all"):



What you will see now is just two contacts! This is because you are logged in as admin, and the third contact created by the other user before was private. So what you see is a contact you can change ("first contact") and another one you are only allowed to read ("second contact").

Users and Groups:

[top]

Overview:

  • Every user has (exactly) one default group.
  • Every user is member of zero or more groups.
  • Every user is admin of zero or more groups. If the user is admin of a certain group, he has to be member of that group as well.
  • Some users can have a special status, they are superadmins. If you are superadmin, you can do (nearly) everything.

Description:

In order to organize your staff or manage the employees of your organisation, you have to create different groups (you might think of roles as well). There might be a group called 'CEOs', 'human resources', 'secretary' or anything else.

Every single contact or any other piece of shared information will be assigned to a certain group, and members of this group can access this information.

The more groups you are member of, the more information you have access to. In a hierachically organized company, the boss will be member of all groups, and other employees might be member of a single group only - maybe the only member of this group.

The default group:

Every user has a default group assigned to him or her. This default group is used whenever leads4web has to decide which group to use if there is no additional information.

Being admin of a group:

If a user is admin of a group, he has additional rights concerning this group. (...)

Being member of no group at all:

If a user does not belong to any group, he cannot access the system any more.

Permissions:

Adding Users: Only superadmins are allowed to add users.
Changing Users: A superadmin can change the membership of any user, a user himself can change his personal information.
Deleting Users: Only superadmins are allowed to delete users.

Examples:

A single user:

Not a very interesting case. Leads4web/3 is to manage shared information about contacts, companies, follow-ups and so on. So there should be more than one user. Sure, you can use a single-user installation; but this chapter might not be of any use to you.

A small company:

Suppose there is a company with, say, 5 leads4web users, 2 of which are chiefs, and 3 sales forces.

Use "Configuration -> User Management" or "Configuration -> User Management II" if you want to create those users:



As you can see, Chief1 and Chief2 both are members of the group "small company" as well as of the group "Sales"; therefore they have access to any information of those groups. The three salesman on the other hand have access to the information defined within group "sales".

It's getting a bit complex, isn't it?

Opening the entry for Chief1 yields to the collection of groups this user is member of - with one exception: As you can see in the first level, Chief1 is member of the group "small company" - this membership will not be repeated in the next level, as the tree would get much to complex this way.

Modules:

[top]

Overview:

  • Modules let you personalize extended features of leads4web.
  • Modules can be accessed only by users who have been authorized.
  • There are various preinstalled modules, like Module Admin.

Module Admin:
This module is used to authorize users to access the modules, including this module itself. If you select an user, you can permit or restrict the accessability of any of the available modules.

Document Management:
This module lets you manage documents which can be used as templates in combination with the data managed in leads4web.

Add folder

You are allowed to add folders if you meet one of these conditions:
  • you are superadmin
  • you are in the root level and you are admin of at least one group
  • if you are not in the root level, you must be admin of the parents group
