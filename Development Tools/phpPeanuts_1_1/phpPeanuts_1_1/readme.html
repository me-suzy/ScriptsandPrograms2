Several bugs and notifications fixed, see the list on the bugs section of the website

In 1.1 beta 1 Since phpPeanuts 1.0a:

  1. Null values can now be stored for number, date, time and timestamp datatypes. The user interface framework has allways converted empty strings entered by the user into null values and set those on the corresponding properties of the updated or created object, but now the persistancy framework really tries to store these null values into the database. If the columns in which these null values are to be stored are specified als NOT NULL, query errors will be the result. To prevent this you need to check all table definitions of existing applications to match with settings for the minLength of the corresponding propertyDescriptors: Where null is not allowed, minLength should be set to a number larger then 0. This will make the framework report a proper error to the user and let the user correct the value he entered or selected. See how to convert and validate user input. Where null should be allowed, the database table should be modified accordingly. Be aware that MySql versions prior 3.23.2 can not index columns with null values. See problems with null values in the MySql documentation. If do not (allways) want the framework to convert empty strings to null values, you may override the convert method on class StringConverter. If this solution does not work for you, please ask for alternatives on the forum.
  2. Select and Dialog widgets now will prevent the removal of a value if the corresponding propertyDescriptor or id-propertyDescriptor's minLength is larger then 0. In older releases minValue was used for this, but that was not consistent with form validation. You may have to check propertyDescriptors in exesting applications for this in order to get the expected behavior with the new version of the framework,
  3. Sql files for the examples now create tables according to the properperty descriptors in the new examples code. When upgrading exisiting examples you should re-run the sql files needed for those examples.
  4. HorizontalTablePart that lists peanuts horizontally (one peanut per column) and properties vertically (one property per row),
  5. SearchPage now includes support for the application of a global filter, see the new version of example 9
  6. Polymorphic persistancy, see example 12
  7. Support for multi value properties in navigational queries,
  8. No more notifications triggered by the framework and the examples
  9. Support for time datatype
  10. ErrorHandler now properly handles errors in the ErrorPage, thus avoinding endlessly looping
  11. ErrorHandler level for logging can now be set seperately
  12. classes/dateFunction.php is no longer included.

Known bugs

  1. Column mappings may not be consequently applied. If you need to set a columnName different to the propertyName on an PropertyDescriptor, test your applications database persistence functionality thouroughly.
  2. Reference Anomalies: Before the generic workaround was applied, use of references by phpPeanuts occasionally made uninitialized php variables or associations arbitrarily hold values that seemed to come from another variable. This rarily happened and since the generic workaround it never happend again, so we recon this issue closed. However, it is impossible to guarantee that the problem has been eliminated once and for all. Do not use phpPeanuts for mission critical applications.
  3. Does not work with php5. An upgrade to a portable version functionally equal to 1.1 beta 2 is available.
