Setter Methods

Setter methods are the methods of your generated classes that you will use in setting object properties. By default MyObjects gives setFieldname style names to these methods. The method names can be overriden in the database schema file. A seperate method for all your database table fields are generated by the code generator. Using the getter and setter methods provide encapsulation of the data that is stored on your objects.

Example Method names:

Field Name Field Type Getter Method Name Setter Method Name
userid int (auto_increment) getUserid setUserid
username varchar(16) getUsername setUsername
email varchar(255) getEmail setEmail
admin enum(’Y’, ‘N’) (binary) isAdmin setAdmin
active enum(’Y’, ‘N’) getActive setActive

MyObjects code generator generates validation code for the object properties in setter methods depending on the constraints that you define in your database schema file.

Allowed Constraints For The Database Fields

The constraints that you can add for an object property depend on the corresponding MySql field type in the database table, but some of them apply to all kind of fields.

Global Constraints For All Type Field Types

Some constraints are allowed to be added for all type of fields. These are:

  • Required Constraint
  • Unique Key Constraint

If the required constraint of an object property is set, before the database operations (insert, update) the value for that property is checked. If it is not set, then a MapableNotValidException is thrown.

If the unique key constraint is added for an object property, the existence of the supplied data in the database is checked in the generated setter method. If the data exists in another row of the database, a UniqueKeyExistsException is thrown.

Here is a sample setter method with Unique Key Constraint: 

/**
* Sets User Name
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for User Name
* @throws UniqueKeyExistsException Thrown if the supplied value exists
* as a User Name property of another User object
* @param string $username User Name
* @return boolean Returns true if User Name is set successfully
*/
public function setUsername($username) {
    // Check for validness
    if(!StringValidator::isWord($username, 3, 32)) {
        throw new InvalidValueException('username');
    }
 
    // Check if the unique key username exists
    $result = User::select(array('username'=>$username), 'username');
    if($row = $result->fetch_row()) {
        if($this->username != $row[0]) throw new UniqueKeyExistsException('username');
    }
 
    // Assign the value
    $this->username = $username;
    return true;
}

Constraints For Numeric Field Types

The numeric field types of MySql server are:

  • TINYINT
  • SMALLINT
  • MEDIUMINT
  • INT
  • BIGINT
  • FLOAT
  • DOUBLE
  • DECIMAL

If you are not sure of the type of MySql field you can use generic NUMERIC field type too.

Unsigned Constraint

If a numeric field is set as unsigned, the setter method checks if the assigned value is greater than zero; otherwise it throws an InvalidValueException. A sample setter method with unsigned constraint: 

/**
* Sets Field
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for Field
* @param int $Field Field
* @return boolean Returns true if Field is set successfully
*/
 public function setField($Field) {
    // Check if the value is unsigned
    if($Field < 0) {
        throw new InvalidValueException('Field');
    }
    // Assign the value
    $this->Field = $Field;
    return true;
}

Minimum Value Constraint

A minimum value for a numeric object property can be set using the Minimum Value Constraint. The generated setter method checks the assigned value if it is less than the specified minimum value, if the supplied value is not valid an InvalidValueException is thrown by the setter method.

Maximum Value Constraint

A maximum value for a numeric object property can be set using the Maximum Value Constraint. The generated setter method checks the assigned value if it is greater than the specified maximum value, if the supplied value is not valid an InvalidValueException is thrown by the setter method.

Constraints For String Field Types

The string field types of MySql server are:

  • CHAR
  • VARCHAR
  • TINYBLOB
  • BLOB
  • MEDIUMBLOB
  • LONGBLOB
  • TINYTEXT
  • TEXT
  • MEDIUMTEXT
  • LONGTEXT

If you are not sure of the type of MySql field you can use generic STRING field type too.

Restriction Constraint

You can add restrictions to your string fields about the pattern of the string that they will contain. There are some builtin patterns that you can use in MyObjects. These patterns are:

  • date - The string must be in valid date format. (YYYY-MM-DD)
  • time - The string must be in valid time format. (HH:MM:SS)
  • datetime - The string must be in valid datetime format. (YYYY-MM-DD HH:MM:SS)
  • year - The string must be in valid year format. (YYYY)
  • email - The string must be a valid email address.
  • cleanText - The string must be formed of alphabetic characters, space character, &acute; character and ' character. (Without digits)
  • word - The string must be formed of alphabetic characters, digits or _ character. No white space is allowed.
  • alpha - The string must be formed of just alphabetic characters.
  • numeric - The string must be formed of just digits.

A sample setter method with email constraint: 

/**
* Sets Email of the user
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for Email of the user
* @param string $email Email of the user
* @return boolean Returns true if Email of the user is set successfully
*/
public function setEmail($email) {
    // Check for validness
    if(!StringValidator::isEmail($email, 0, 255)) {
        throw new InvalidValueException('email');
    }
 
    // Assign the value
    $this->email = $email;
    return true;
}

Minimum Length Constraint

A minimum length for a string object property can be set using the Minimum Length Constraint. The generated setter method checks the assigned strings length if it is less than the specified minimum length, if the supplied string is not valid an InvalidValueException is thrown by the setter method.

Maximum Length Constraint

A maximum length for a string object property can be set using the Maximum Length Constraint. The generated setter method checks the assigned strings length if it is greater than the specified maximum length, if the supplied string is not valid an InvalidValueException is thrown by the setter method.

A sample setter method with length constraints: 

/**
* Sets Password
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for Password
* @param string $password Password
* @return boolean Returns true if Password is set successfully
*/
public function setPassword($password) {
    // Check for validness
    if(!StringValidator::isLengthValid($password, 4, 32)) {
        throw new InvalidValueException('password');
    }
 
    // Assign the value
    $this->password = md5($password);
    return true;
}

Regular Expression Constraint

If you want to check the assigned string against a regular expression pattern in the setter method, you can add a regular expression constraint.

Constraints For Enum and Set Field Types

Enum and Set field types can not have the values that are not defined in the MySql table. The value that is being tried to assigned is controled in the setter method. Here is an example setter method: 

/**
* Sets Continent of the country
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for Continent of the country
* @param string $Continent Continent of the country
* @return boolean Returns true if Continent of the country is set successfully
*/
public function setContinent($Continent = 'Asia') {
    switch($Continent) {
        case 'Asia':
            $this->Continent = $Continent;
            return true;
        case 'Europe':
            $this->Continent = $Continent;
            return true;
            .
            .
            .
            .
        case 'South America':
            $this->Continent = $Continent;
            return true;
    }
    throw new InvalidValueException('Continent');
}

Constraints For Time Field Types

Time field types DATE, TIME, DATETIME, YEAR are automatically checked against a regular expression pattern in the setter methods.

Store Functions

Store functions are the functions that will be called in setter methods before assigning the suplied value to an object property. You can think these functions as filter functions which modify the data before writing it to a database. For instance many applications store passwords in databases as a md5 hash string. Setting the md5 function as a store function in a password field generates a setter function like the one below: 

/**
* Sets Password
*
* @throws InvalidValueException Thrown if the supplied value
* is not valid for Password
* @param string $password Password
* @return boolean Returns true if Password is set successfully
*/
public function setPassword($password) {
    // Check for validness
    if(!StringValidator::isLengthValid($password, 4, 32)) {
        throw new InvalidValueException('password');
    }
 
    // Assign the value
    $this->password = md5($password);
    return true;
}

Here first the supplied data is checked for validness. If it is valid then the supplied parameter value is sent to the md5 function, then the returning hash of md5 is assigned as the password object property.

Validation Functions

Sometimes you may want to create your own validation functions for your object properties. The validation function that you’ve defined is called to validate the supplied parameter in a setter function.

Default Values

You may add default values for your object properties. These values will be assigned in the constructor of your objects and they will also be added in your setter methods parameters as default values.

Foreign Keys

If you add foreing key fields to your database schema, in addition to the setter method for that key, another setter method for the foreign object will be created in your classes.

Here is an example taken from a class named City: 

/**
* Sets the foreign Country object
*
* @param Country $country Foreign Country object
* @throws InvalidValueException Thrown if the primary key value
*  of the foreign object is not set
* @return boolean Returns true if the foreign object is set successfully
*/
public function setCountry(Country $country) {
    if($country->Code != null) {
        $this->country = $country;
        $this->CountryCode = $this->country->CountryCode;
    } else {
        throw new InvalidValueException('country');
    }
    return true;
}

In this example CountryCode property of the City class is the foreign key of Country class. The generated setter method accepts a Country object and sets it as the pointer to the Country object in the class. In addition it updates the CountryCode property of the City object with the supplied Country objects primary key property (Code in this case).
