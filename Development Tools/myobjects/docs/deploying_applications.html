Deploying Applications

After designing your database schema and creating your classes to a folder, you will notice that a file named MyObjectsSettings.php is created. Edit this file and set the path for your runtime classes and the generated classes. Additionally set the database access information about your MySql server. 

// Runtime library path. This is the path of the directory named 'runtime'.
// Without a trailing slash
define('MyObjectsRuntimePath', 'YYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYY/AAAAA');
 
// The path of generated classes
// Without a trailing slash
define('MyObjectsClassPath', 'YYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYYY/XXXXXX');
 
/**
* MyObjects Runtime Settings
*
* This class defines some configuration constants
*
* @author Erdinc Yilmazel <erdinc@yilmazel.com>
* @version 1.0
* @package MyObjectsRuntime
*/
class MyObjectsRuntimeSettings {
    
    // This Section Should Be Edited -------------------------------------------
    
    // Database Host Name
    const DB_HOST =             'localhost';
    // Database User Name
    const DB_USER =             '';
    // Database Password
    const DB_PASSWORD =         '';
    // Database Name
    const DB_NAME =             'casestudy';
    
    // End Of Editing Section. Do not change anything below this unless you know
    // what you are doing.
}

In the scripts that you are going to write you should include MyObjectsSettings.php and Base.php files. Here is a header that you can add in your php scripts: 

require_once('MyObjectsSettings.php');
require_once(MyObjectsRuntimePath . '/Base.php');
 
// Write your code below this.
