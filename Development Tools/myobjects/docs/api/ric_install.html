Requirements
--------------------------------------------------------------------------------
The requirements of MyObjects are:
Php 5 with DOM, MySqli, Perl-Compatible regular expressions support
MySql server. (Version > 4.1 is needed to use prepared statements support)

Installation
--------------------------------------------------------------------------------
To install the application just extract the archive to a web accessable
folder. If you want to use the webclient, webclient/tmp folder
should be writable by the web server.

Use the following command on linux/unix platforms to make it writable.
$ chmod 777 myobjects/webclient/tmp

To access the web client just open your favorite web browser and head to the
folder that you made the installation.

You can also use the command line client MyObjects.php to access some
of the functionalities of the application. ($php MyObjects.php)

Upgrade
--------------------------------------------------------------------------------
To upgrade from a previous version overwite all the existing files and re-generate
your classes.

How to get started with MyObjects
--------------------------------------------------------------------------------
To use MyObjects as a MySql - Php 5 objects bridge you first need to generate a
schema file describing your MySql database or databases. You can create a schema
file from your existing databases using the WebClient or Command Line Client (MyObjects.php)
of MyObjects. Additionally you can create the schema file from scratch using the WebClient
or you can even create it manually. The database schema files are xml files that are in
Database Description Language format that is described in ddl.xsd Xml schema file.

Once you created your database schema, added constraints about your table fields and
relations about your database tables you can use MyObjects Code Generator to compile your
database schema file to Php 5 classes. The code generator creates a seperate class for
each of your database tables, adds required validation code for your fields and adds some
useful methods regarding your database relations.

As a programmer you can use the generated classes to map database records to Php objects,
Php objects to database records, Xml files to php objects and Php objects to xml files.
MyObjects has many handy classes for you to do many tasks in an efficient and easy way.
You can see the api documentation to get used to the runtime classes that MyObjects provide
and you can examine the example applications in the example-app folder.

For more detailed information about using MyObjects please visit it's website at
http://www.myobjects.org
