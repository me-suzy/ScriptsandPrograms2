Changes

# Release Candidate 2 (Hopefully the final release candidate)

  * 2004-12-07
    * Date/time validation functions in generated classes fixed (Erdinc)
    * Import schema sql to a MySql server functionality added (Erdinc)
    * A new method for loading array values into Mapable objects Mapable::loadArray() added (Erdinc)
    * Super classes for the generated classes can now be defined (Erdinc)

  * 2004-11-30
    * Class generator does not overwrite MyObjectsSettings.php file anymore. (Erdinc)
    * The foreign keys are no longer registered automatically by default. (Erdinc)
    * Getter methods for foreign objects automatically register the foreign object if it
    wasn't registered correctly before (Erdinc)

  * 2004-11-28
    * Auto type time fields are now handled correctly. (Erdinc)
    * The setter methods of unique fields now have a second paramether that enables
    skipping of unique value existence check. (Erdinc)
    * In the get() method of generated classes, ObjectNotFoundExceptions are automatically
    caught while registering the foreign keys. It will no longer give an ObjectNotFoundException
    if the foreign key of an object does not exists. (Erdinc)

  * 2004-11-26
    * DebugMode which enables logging of the executed sql queries was added. (Erdinc)

  * 2004-11-23
    * Added a new icon for required fields at the table editor. (Erdinc)

  * 2004-11-20
    * Removed Options menu from the web client. The options are now set
    * on the class generation screen. (Erdinc)
    * Changed error handling
  
  * 2004-11-19
    * DbModel::get() method was modified and the second parameter was changed.
    Now the second parameter is a boolean flag for auto registering the foreign
    key objects automatically. The old second parameter $compact was moved to the
    third position because it will be used less likely. (Erdinc)
    (Thanks to Jay Kramer for the idea)
    * Fixed character encoding in ddl files (Markus)

  * 2004-11-18
    * Fixed a few bugs in StringValidator. (Erdinc) (Thanks to Jay Kramer)
    * Fixed "Add new field" links in the Tree menu (Erdinc)
    * Added control for 0 number of rows in DbModel::get method (Erdinc)

  * 2004-11-15
    * Added possibility to use different images in dynamicTree (Markus)
    * The schema menu now displays the types of the fields (Erdinc)

  * 2004-11-14
    * Minor Gui changes (Erdinc)

  * 2004-11-13
    * Options dialog settings now depends on the databasename (Markus)
    * Added "MySql-Connection Settings" to options dialog (Markus)
    * Added "Change Database Options" - button to editDatabase template (Markus)
    * Added class attribute to the first row in editDatabse template (Markus)
    * Changed class attribute in editfield template to start with row1 (Markus)
    * Changed class attribute in edittable template to start with row1 (Markus)

  * 2004-11-12
    * Import/Export functionality for individual tables and databases was added.
      (Erdinc)
    * Added new Navigation-Node "Sql" and moved the Node "Generate Sql Dump" to it
      (Markus)
    * Moved Node "Generate Schema From Existing Database" to the last position
      (Markus)
    
  * 2004-11-08
    * A bug that caused the retreival of information about enum and set fields
    incorrectly was fixed. The web client now expects the enum and set values to
    be input as 'value1', 'value2' style. (Erdinc)

  * 2004-11-06
    * Tab indexes were added in the web client forms. (Erdinc)

  * 2004-11-05
    * Links at the web client that were pointing to localhost were fixed. (Erdinc)
    * Work folders that are older than one week are now deleted automatically. (Erdinc)
