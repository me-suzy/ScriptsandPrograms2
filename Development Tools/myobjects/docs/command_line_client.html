MyObjects Command Line Client

You can use the MyObject command line tool for class generation and automatic database schema generation. MyObjects command line client is the MyObjects.php file that is stored in the installation folder of MyObjects. On the command line type #php MyObjects.php to see the available parameters: 

MyObjects Command Line Client v1.0

You can perform several tasks using a project file or by passing command line op
tions.

Note that in database schema generation MyObjects Command Line Client
always assumes default options for every field and can not
automatically determine foreign keysfrom the database tables.
These kind of controls, such as foreign keys and validation controls
should be added manually or you can rather use the MyObjects Web Client
for more advanced features.

Usage: php MyObjects.php -p <projectFile>
       php MyObjects.php [options]

  -projectfile, -pf <Project File>      Specify a project file

  -generateclasses, -gc                 Perform class generation task
    -schema, -s <Database Schema>       Specify database schema file
    -out, -o <Output Dir>               Specify output directory
    -author, -a <Author Name>           Specify author name
    -package, -pck <Package Name>       Specify package name
    -version, -v <Version Numer>        Specify version number
    -copyruntime, -cr                   Copy the runtime classes to output dir
    -nogetters, -ng                     Don't generate getter functions
    -nosetters, -ns                     Don't generate setter functions
    -nocomments, -nc                    Don't generate phpdoc comments

  -generateschema, -gs                  Perform database schema generation task
    -schemaout, -so <Output Schema>     Output schema file
    -hostname, -h <Database Host>       Database host name
    -user, -u <Database User>           Database user name
    -password, -pa <Database Password>  Database password
    -databases, -d <Databases>          Database names seperated by space
