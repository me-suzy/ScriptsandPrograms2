Getter Methods

Getter methods are the methods of your generated classes that you will use in accessing object properties. By default MyObjects gives getFieldname style names to these methods. The method names can be overriden in the database schema file. A seperate method for all your database table fields are generated by the code generator. Using the setter and getter methods provide encapsulation of the data that is stored on your objects.

Example Method Names:

Field Name Field Type Getter Method Name Setter Method Name
userid int (auto_increment) getUserid setUserid
username varchar(16) getUsername setUsername
email varchar(255) getEmail setEmail
admin enum(’Y’, ‘N’) (binary) isAdmin setAdmin
active enum(’Y’, ‘N’) getActive setActive

Use of Getter Methods

Getter methods do not manipulate the data before returning it. They simply return the value stored in the corresponding object property. There is an only one exception in getter methods that is in Binary Enum fields. When a table field is marked as binary enum the generated getter methods are by default named as isFieldname and they return boolean true or false values instead of the value that is stored in the object property.

Here is an example: 

$usr = User::get(15); // Get the User object with userid = 15;
// admin field is binary enum. If the admin property of $usr object is Y then this method
// returns true, if it is F it returns false.
if($usr->isAdmin()) {
    echo $usr->getUsername() . " is admin";
}
 
// active field is a regular enum. If the active property of $usr object is Y then this method
// returns Y, it it is F it return F.
if($usr->getActive() == 'N') {
    echo $usr->getUsername() . " is not active";
}
