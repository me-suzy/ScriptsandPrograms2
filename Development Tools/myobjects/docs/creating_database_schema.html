Creating Database Schema

You can create a database schema file in several ways.

  • Using the command line or web client to generate it from existing databases
  • Designing it using the web client
  • Creating it on your own using a text editor

How to generate a database schema file from your existing MySql databases

Using the Web Client to create a database schema file

To create a database schema file using the MyObjects WebClient you need to open your web browser and go to the MyObjects main page. Using the toolbar menu or the link that is on the left side of the page go to the myobjects/?generateSchema page.

You should note that MyObjects does not store any user/password information. (For keeping the WebClient simple and for security.)

On the Schema generation page you will see a form that you should enter database access options. Fill in this form and press the Generate Schema button. MyObjects will connect to your existing database, retreive information about the tables and fields and it will create a database schema file for you.

MyObjects does not make assumptions about the relations in your database or type of information that you will store in your database tables. Using the Command Line Client to create a database schema file

MyObjects has a command line based tool that you can use in schema generation and code generation. The Command Line Client is a script that is also written in Php language. This script is MyObjects.php file that you can find in the installation folder. On the command prompt when you type: $ php MyObjects.php you will get the list of options that you can provide to the MyObjects Command Line Client.

To generate a database schema file you should provide -gs (Generate Schema) option to the script. You can also use -generatechema option that is an alias to -gc. Additionally you should provide database access information as command line options. These options are :

  • -hostname, -h <Database Host Name> (-h localhost)
  • -username, -u <Database User Name> (-u root)
  • -password, -pa <Database Password) (-pa yourdatabasepassword)
  • -databases, -d <Database Names) (-d databaseName1 databaseName2 …)

You can also specify the output schema file name using the -so option (or -schemaout). If you do not provide the output file name the default name schema.xml will be assumed. Here is an example usage that will generate database schema of databases name world and spring : 

$php MyObjects.php -gs -h localhost -u root -pa 1234 -d world spring -so world.xml;

You can also create a project file and perform several tasks at one time. MyObjects Command Line Client project files are defined in the project.xsd Xml schema file.

To perform the same operation using a project file you need to create an Xml file as below : 

<?xml version="1.0"?>
<project name="World">
 <generateSchema output="world.xml">
  <connect host="localhost" user="root" password="1234">
   <database>world</database>
   <database>spring</database>
  </connect>
 </generateSchema>
</project>

After creating the project file you can pass the -p parameter to the MyObjects command line client. 

$ php MyObjects.php -p project.xml
