MyObjects Runtime

MyObjects defines a set of classes for the developers to use with the generated classes. The runtime classes provide database operations and xml operations smoothly.

Core.php File

Mapable Interface

MyObjects code generator generates classes which implement the Mapable interface. Most of the runtime classes work with the objects which implement the Mapable interface. For detailed documentation of the methods that are defined in Mapable interface see the Api Documentation.

DbModel Class

DbModel Class defines some methods for inserting, updating the object data in MySql server, retreiving data from MySql server to your Objects and removing object records from the database. Inorder to use your generated classes with MySql server, you need to get used to the methods of this class. The Api Documentation for the DbModel class will guide you in the development phase.

The methods of DbModel class work with the objects of these classes which are instances of Mapable.

int countSimilar() Counts the number of similar objects in the database that have the same properties with the specified object
boolean delete() Removes the object from the database
boolean deleteSimilar() Deletes the similar objects which have the same properties as the specified object
boolean exists() Checks the database for the existance of the object

mysqi_result selectDifferent() Selects the objects that have different properties than the supplied object

For the detailed descriptions of the methods and for example codes, please refer to the Api Documentation

XmlModel Class

XmlModel class defines some methods for serializing the Mapable objects into XML files, XML strings, DOM objects and creating Mapable objects from the same resources.

The Api Documentation for the XmlModel class will guide you in the development phase.

The methods of XmlModel class work with the objects of these classes which are instances of Mapable.

The created XML files are in the format that is defined in mapable.xsd Xml Schema file.

Base.php File

Base.php file should be included by the scripts that will use MyObjects. Here is a header that you can use in your Php scripts: 

<?php
// Include the settings file which contains database access information
require_once('MyObjectsSettings.php');
// Include MyObjects base
require_once(MyObjectsRuntimePath . '/Base.php');
?>

MyObjectsBase Class

MyObjectsBase class implements the Singleton design pattern and can only have one instance. This object instance opens database connections when needed by DbModel class or it can return the DatabaseConnection instance if there is a database connection available.

Method Summary For the MyObjectsBase Class

Return type Method Name Method Description
void destruct() Closes the database connections
void dbConnect() Establishes a database connection
DatabaseConnection getDbConnection() Returns the available database connection
float getElapsedTime() Returns the elapsed time since the MyObjectsBase object was created
MyObjectsBase getInstance() Returns the MyObjectsBase instance
boolean isConnected() Checks whether there is a valid database connection

To create or get a MyObjectsBase instance simply call the MyObjectsBasee::getInstance() method. If there is already a MyObjectsBase object created, it will be returned, otherwise a new instance will be created and will be returned. Becase the constructor for MyObjectsBase is private you can not write $base = new MyObjectsBase(); in your php scripts. 

$base = MyObjectsBase::getInstance();

If you don’t pass a parameter for the getInstance method and if it is called for the first time, the method will attempt to connect to the database server with the parameters that are defined in the MyObjectsSettings.php file. If you want to override these values, you can create a MyObjectsConfig instance and pass it as the first parameter to the getInstance() method. 

//MyObjectsDefaultConfig class implements the MyObjectsConfig interface
$config = new MyObjectsDefaultConfig('localhost','username', 'password', 'database name');
// Use the created config instance
$base = MyObjectsBase::getInstance($config);

If you want to get a MyObjectsBase instance, but you don’t want it to create a database connection, then pass the second parameter of getInstance() method false. 

$base = MyObjectsBase::getInstance(null, false);

When you need a mysqli object for database access you can call the getDbConnection() method from the MyObjectsBase instance. 

// Get MyObjectsBase instance
$base = MyObjectsBase::getInstance();
$db = $base->getDbConnection();
$result = $db->query("Select * from books limit 5");

DatabaseConnection Class

DatabaseConnection class extends the mysqli class and it is used in entire database operations. This class can be replaced by some other classes which implement the ext/mysqli api to support other databases but it is currently not supported nor encouraged.

View.php File

View Interface

The View interface should be implemented by the classes that will provide views for Mapable objects. A view is simply the representation of a portion of data that is stored in a Mapable object. Every Mapable object defines a method named getView(View $view) that you can use.

The methods defined in View interface are:

  • void setModel(Mapable $model)
  • string toString()

DefaultMapableView Class

Default view Class that demonstrates how a View class can be implemented. This class simply shows every object property in an HTML table.

Example Usage: 

$user = new User();
$user->setName('Erdinc Yilmazel');
$user->setEmail('erdinc@yilmazel.com');
$view = new DefaultMapableView();
 
// Echoes an Html table
echo $user->getView($view);

XmlView Class

XmlView class is another class that implements the View interface. It simply returns an Xml string that is generated by the XmlModel class.

Example Usage: 

$user = new User();
$user->setName('Erdinc Yilmazel');
$user->setEmail('erdinc@yilmazel.com');
$view = new XmlView();
// Echoes the generated xml string
echo '<pre>' . $user->getView($view) . '</pre>';
