MyObjects Documentation

MyObjects is a set of tools for class generation, database schema generation and object persistence for Php 5 and MySql. It is divided into two sub projects, a code generator for class generation and runtime classes for object mapping. Objects can be mapped to MySql tables and XML files, and can be created from database and xml sources easily.

A key feature of MyObjects is generating validation code in your classes depending on the constraints that you have defined. Here is an example code that you can write after creating a class named User in MyObjects: 

try {
    $usr = User::get(15); // Get the User object with id = 15 from the database
    $usr->setEmail('some invalid email'); // Throws an InvalidValueException
    DbModel::update($usr); // Does not reach this step
} catch (ObjectNotFoundException $e) {
    die('User with id = 15 could not be found');
} catch (InvalidValueException $e) {
    die("Error :" . $e->getMessage() .' is invalid');
}

Features

  • Automatic database schema generation from existing databases
  • Web based database schema generation from scratch
  • Command line based database and class generation
  • Generation of Sql statements from the designed database schema
  • Generation of Classes that will map to the database tables. (A seperate class for each table is created)
  • Full validation code through setter methods in generated classes
  • phpDoc style comments in generated classes
  • Minimizes runtime operations by providing many features in generated classes
  • Xml serialization of the objects of generated classes and deserializing of the objects from Xml files
  • Inplemented in new Php 5 object model
  • Uses the new MySqli extension for php to support new features of MySql
  • Uses the new libXml based DOM extension to provide generation, parsing and validation of database schema files. (Database Description Language Files (DDL))

Requirements

  • Php 5
  • DOM Extension for Php 5
  • Improved MySql Extension for Php 5
  • Perl Type Regular Expressions Support
  • MySql server (Version > 4.1 is needed for prepared statements support)

Api Documentation

MyObjects has a full api documentation for the developers who want to get used to the runtime classes and for the people who want to extend MyObjects. You should especially be familiar with the DbModel and XmlModel classes.
