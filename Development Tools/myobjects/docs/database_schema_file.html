Database Schema File

Database Schema File is an Xml file that is defined with Database Description Language xml schema file. Database Schema File describes the databases, tables, fields of your MySql server. Additionally it contains information about the constraints that you want to add about your table fields.

The database schema file can be created in following ways

  • Using the Web Client gui you can design the database schema.
  • Web Client can create the database schema file for you from your existing MySql databases
  • Command Line Client can create the database schema file for you from your existing MySql databases
  • You can create the database schema file yourself using a text editor. (You must obey the rules that are defined in Database Description Language file format.

For the detailed information about how to create database schema files, how to add constraints about your table fields please see the Database Description Language specification.

Usages of database schema file

  • MyObjects Code Generator creates Php 5 classes for the tables that are defined in the database schema file. It also looks at the constraints that you have defined in the schema file to create validation code for your table fields.
  • MyObjects Web Client can generate sql code from a database schema file that you can run on a MySql server to create the databases defined in your database schema file.
  • It can be used to create web forms for your database tables. A new project that is currently in planning phase called MyObjects Explorer will use this technique.
