MANAGING THE SYSTEM SETUP

Overview

  1. Click Setup on the menu bar.
  2. Change the system settings as required. These are explained below.
  3. Click the Save Changes button.

When you click on Setup (Step 1), you will see the System Setup screen where you can edit various properties of your Autoresponse Plus system. Most of them can be changed in this screen. The first two can only be changed by editing the file 'paths.pl' which can be found in the Autoresponse Plus scripts directory. this is usually .../yourdomainorwhatever/...etc. etc/cgi-bin/arplus.

  • Data file path

This is the absolute path to the directory on your server where Autoresponse Plus stores it's data. The directory cannot be changed here. To change the directory, you need to edit the file 'paths.pl' which you will find in your Autoresponse Plus scripts directory. 'paths.pl' is normally edited only during the installation procedure.

  • Your web site URL

This is the address of the web site where Autoresponse Plus is installed. You can change this by editing the file 'paths.pl' which you will find in your Autoresponse Plus scripts directory. 'paths.pl' is normally edited only during the installation procedure.

  • Your domain

This is the domain where Autoresponse Plus is installed. It should be in the simple format yourdomain.com. There should be no http:// or www or /index.html etc.

  • URL of your Autoresponse Plus scripts directory

This will be something like http://www.yourdomain.com/cgi-bin/arplus. There should be no slash at the end.

  • Path to your sendmail program

Enter here the absolute path to your server's sendmail program. This is usually something like /usr/bin/sendmail. If you are not sure what to enter here, contact your host's support desk for assistance.

  • Your support e-mail address

Enter the address of your own support desk. This is the address which your contacts can send e-mail to if they need help when using your Autoresponse Plus system. Error pages that your contacts may see have a link to this e-mail address.

  • System e-mail address

    This e-mail address is used for two purposes:

    • To send out e-mailed confirmations when a contact unsubscribes from an autoresponder.
    • To send out one-off e-mails generated by the 'Batch E-mail' option on the subscriber list screen.
  • Show news banner?

    Check this box to display the current AutoResponse News banner. The banner shows details of updates and other AutoResponse Plus news. Click on the banner for further information on any of the topics shown.

  • Use file locking?

We strongly recommend you check this box to prevent data corruption.

  • Login cookie life

On the Autoresponse Plus login screen there is a check box which says 'Remember Next Time?'. When you check the box, your owner name and password will automatically be completed the next time you visit the login screen. This value is the number of days the system will remember your details until you have to enter them again. Set this value to zero (0) if you want your details to be remembered only until you close your browser at the end of the current session.

  • E-mail failures

The system automatically handles bounced or undeliverable messages. This value is the number of failures that will be allowed for each subscriber. When this number is exceeded, the subscriber's status will be changed to 'Failed' and no more messages will be sent.

  • Number of subscribers per page

This is the number of subcribers that will be shown on a single screen on the subscriber list.

  • Force default personalization?

If you are using personalization tags in your messages, check this box to use the default values in the tags instead of your subscriber's details. For example, if you message contains {FIRSTNAME Internet_Marketer} then every recipient will see "Internet Marketer" and not their first name.

  • Show sub menu bar tool tips?

When you move your mouse over the options in the sub menu bar, a pop up will tell you what each option does. Check this box to enable these popups and uncheck it to switch them off.

  • Custom success and failure pages (There are four)

When contacts subscribe or unsubscribe to your autoresponders they will see the default confirmation pages which are built into the system. You can create and use your own default custom pages if you wish. Enter their URL's here. Custom pages can also be defined for each tracking tag so that you can use different custom pages depending on the source of the subscription. The URLs for these pages are entered in the Tracking Tags section.

  • Affiliate link text

When the "Include Affiliate Link?" check box is checked in the page setup of an autoresponder, each message will have your affiliate link at the end. The default link text is Powered by Autoresponse Plus. This text can be changed here. You can also include the link text anywhere else in messages, headers, footers, subjects and signatures using the tag {AFFILIATE_LINK_TEXT}. The link itself can be included using the tag {AFFILIATE_LINK}.

  • Anti-spam message

This can be added to your follow-up and one-off messages with the {ANTISPAM_MESSAGE} tag. It is a message telling you subscriber what to do if they believe they have been subscribed to your autoresponder (or received your e-mail) without their permission.

  • Send e-mail confirmation on unsubscribe?

Check this box to send a confirmation e-mail to your contact when they unsubscribe from an autoresponder.

  • Unsubscribe link text

When the "Include Unsubscribe Link?" check box is checked in the page setup of an autoresponder, each message will have a single-click "unsubscribe" link at the end. The text which is displayed with this link can be changed here. You can also include the link text anywhere else in messages, headers, footers, subjects and signatures using the tag {UNSUBSCRIBE_LINK_TEXT}. The link itself can be included using the tag {UNSUBSCRIBE_LINK}.

  • Unsubscribe confirmation e-mail text

This is the body text of the e-mail message that will be sent to contacts who unsubscribe from an autoresponder. This message will be sent only when the "Send e-mail confirmation on unsubscribe?" box is checked.

  • Signatures (There are three)

Information you enter here can be included in messages using the {SIGNATURE 1}, {SIGNATURE 2} and {SIGNATURE 3} tags. These would be useful for corporate, sales or support signatures, for example.

  • Adverts (There are three)

You can enter ads here which can be included in messages using the {AD 1}, {AD 2} and {AD 3} tags. Ads can include HTML if they will be used in HTML messages.

  • Reserved e-mail addresses

This is a list of addresses which will not be permitted as autoresponder listening addresses. These are normally addresses which are already in use. For example, webmaster@yourdomain.com

  • Ban list

Here you can enter a list of e-mail addresses and/or domains from which autoresponder subscriptions will be rejected.

When you have changed the settings to match your own particular setup, click the Save Changes button.
