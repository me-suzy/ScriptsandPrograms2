CREATING A NEW AUTORESPONDER

Overview

  1. Click on Autoresponders on the menu bar.
  2. Click on Create Autoresponder on the sub menu bar.
  3. Set the properties of the new autoresponder. These are explained below.
  4. Click the Create button.

When you click Create Autoresponder (Step 2), you will see the Autoresponders > Create screen. The settings on this screen define the main properties of your new autoresponder. Most of these properties can be changed later by editing the autoresponder. Here is an explanation of the fields on this screen:

  • From name

This is the name of the person or company that your follow-up messages will appear to come from. When a subscriber receives a message from this autoresponder, they will see this entry as the sender of the message. Valid examples are John Smith, The Widget Company, The Widget Sales Team, The Widget Support Desk.

  • Reply address

When your subscriber receives a message from your autoresponder and decides to reply to you, this is the e-mail address where their reply will be sent. If your contact clicks on their 'Reply' button while reading a message from your autoresponder, the reply will be sent to the address you enter here. This can be an e-mail address at any domain. The reply address cannot be the subscription address of any of your autoresponders because this would cause a loop situation. Valid examples are jsmith@yourdomain.com, sales@yourdomain.com, support@widgetsco.com.

  • Subscription address

Subscribers may subscribe to and unsubscribe from this autoresponder by sending e-mail to the address you enter here. You only need to enter the first part of the address (the part before the @). The domain part (after the @) was entered during the installation and setup process and should not be entered here. Let's say you want to set up a marketing course which will be delivered over five days - a good listening address for this would be course@yourdomain.com. You would enter only 'course' here.

  • Description

The description you enter here will appear in your autoresponder list. This description is for your own use and is not sent out with messages. It allows you to see at a glance what an autoresponder is being used for.

  • Initial status

When you create an autoresponder, it needs to have an initial status. This tells Autoresponse Plus how subscription requests should be handled.

  • Active

    • The autoresponder will immediately be available for subscription, even if you have not finished creating it.

  • Suspended

    • The autoresponder will not be available for subscription. If anyone has already subscribed (highly unlikely at this stage) then no more follow-up messages will be sent to them.

  • Dormant

    • The autoresponder will not be available for subscription. This is the best option to choose here. When you have finished developing and testing this autoresponder, you can set it's status to 'Active' and subscriptions will be allowed.

  • E-mail control

If you would like subscribers to be able to subscribe to and unsubscribe from this autoresponder by sending e-mail to its subscription address, check this box.

  • Form control

If you would like contacts to be able to subscribe to and unsubscribe from this autoresponder by completing a form on a web page, check this box.

When you are finished setting the properties, click the Create button. The new autoresponder will be created and you will be taken to the Autoresponders > Edit screen for your new autoresponder.
