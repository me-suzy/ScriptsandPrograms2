SENDING ONE-OFF EMAILS TO MULTIPLE SUBSCRIBERS

Overview

  1. Click on Subscribers on the menu bar.
  2. Set the filter for the subscribers you want to email (or click Clear Filter on the sub menu bar for all subscribers).
  3. Click on Batch E-mail on the sub menu bar.
  4. Enter the subject and body text for the message you want to send.
  5. Click the Start button.

When you click Batch E-mail, you will see the Subscribers > Batch E-mail screen. Enter the following items then click the Send Now button.

  • From name

For example, your company or e-zine name

  • From e-mail

The reply address for this message.

  • Subject

The subject for this message.

  • Plain text message

The plain text version of the message,

  • HTML message code

The HTML version of the message.

  • Default format

The message format (plain text or HTML) to send when the subscriber record does not contain a preference.

  • Schedule

This field determines when this message will be sent.

  • Next Run
    Once per day (or as often as you have set the cron task to run), Autoresponse Plus sends out any due follow-up messages using a task that runs automatically on your web server. If you choose this option, the message will be sent out the next time the automatic task runs.
  • Monday - Sunday
    The message will be sent out on the day you choose. This is non-repeating and the message will be sent out only once. On the same day the following week, the message will not be sent as the message is a one-off and has been deleted from the queue.
  • Omit unsubscribers

Check this box to exclude everyone who's record has 'User Cancelled' in the status field, regardless of the current filter. This will ensure that you dod not get accused of sending mail to those who have chosen not to receive it.

  • Test only

When you check this box and click the Start button, the plain text and HTML messages you have entered will be sent to your e-mail address. This is defined in the Your Profile screen. No messages will be sent to subscribers as this is purely a test for message format and layout.

When you have clicked the Start button, please wait for confirmation that the sending or queuing process is complete before taking any further action. This could take a few minutes for long lists.
