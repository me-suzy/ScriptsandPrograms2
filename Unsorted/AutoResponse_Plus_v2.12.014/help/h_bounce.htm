BOUNCED E-MAIL MANAGEMENT

Inevitably, some of your follow-up messages will be undeliverable and will be "bounced" back to your server.

Autoresponse Plus has an automatic system for managing bounced e-mails. The system takes into account the fact that although a message did not get through, the problem may have been a temporary fault.

For this reason, you can set how many bounced e-mails will be acceptable for a subscriber before he/she is flagged as 'Failed' and no more attempts made. This value is entered in the System Setup screen which you can reach by clicking Setup on the menu bar.

Your subscriber list shows for each subscriber how many failures have happened so far. You may also see that the subsrciber's status has been changed to 'Failed' because the number of bounced e-mails exceeded the number allowed.

If a subscriber has been flagged as failed, you can change this back to 'Active' by editing the subscriber's details.
