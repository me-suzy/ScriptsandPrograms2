IMPORTING SUBSCRIBER DATA

Overview

  1. Click on Subscribers on the menu bar.
  2. Click on Import on the sub menu bar.
  3. Upload the comma separated variable (CSV) file to be imported by selecting the local data file and clicking Start Import.
  4. Set the import properties and click Create Subscribers to complete the process. The properties you can set are explained below.

You can upload a comma separated variable (CSV) file containing subscriber data. Subscribers and follow-up sequences will be automatically created for each subscriber in the file.

The file must have one record per line. The field may be enclosed in double-quotes ("") although this is not necessary. Here is an example of a valid file structure. Double-quotes ("") are not used in this example.

EXAMPLE IMPORT FILE STRUCTURE
field_1,field_2,field_3,field_4, ..., field_n
field_1,field_2,field_3,field_4, ..., field_n

Click the Browse... button and select the file on your local computer which contains the data to be imported. Click Start Import when you have chosen the file. Please wait for confirmation that the import process is complete before taking any further action.

Note: The file you choose must have a standard DOS 8.3 filename (eg no spaces).

When the import is complete, you will see the Subscribers > Import screen. Here you must set various properties of the import process. Here is an explanation of the properties you can set:

  • E-mail address field number

The e-mail address is the only required field. Enter here the number of the e-mail address field in the imported data. For example, if the subscriber's e-mail address is the first field in a row of data, enter 1. If the first name is the second field then enter 2.

Entries which do not have a valid e-mail address in the stated field will be ignored during the import.

  • First name field number

This value is optional. If the imported data contains the subscriber's first name, enter the number of the field which contains it.

  • Last name field number

This value is optional. If the imported data contains the subscriber's last name, enter the number of the field which contains it.

  • Full name field number

This value is optional. If the imported data contains the subscriber's full name, enter the number of the field which contains it.

  • Format preference field number

This value is optional. If the imported data contains a preference for message format (plain text or HTML), enter the field number which contains it.

  • Plain text format field entry

This value is optional. If the imported data contains a preference for message format (plain text or HTML) and you have entered the field number that holds it, enter here the actual field entry that denotes the preference is for plain text. In other words, what word, letter, number or phrase in the imported data indicates that the subscriber would prefer to receive plain text messages.

  • HTML text format field entry

This value is optional. If the imported data contains a preference for message format (plain text or HTML) and you have entered the field number that holds it, enter here the actual field entry that denotes the preference is for HTML. In other words, what word, letter, number or phrase in the imported data indicates that the subscriber would prefer to receive HTML messages.

  • Default format

If the imported data for a subscriber does not contain a format preference (plain text or HTML), choose here what should format of message should be sent.

  • Autoresponder

Choose here the autoresponder that each of the new subscribers will be automatically subscribed to.

  • Tracking tag

Choose here the tracking tag, if any, that will be associated with each of the new subscribers.

  • Initial status

Choose here the initial status of all the new follow-up sequences that will be created.. The choices are:

  • Active
  • Suspended
  • Finished

When you have completed the import properties, click the Create Subscribers button. If the list is large, this may take some time. Do not take any further action until you receive confirmation that the process is complete. You will be returned to the subscriber list screen and will be told how many subscribers were successfully imported. Any failures usually means that the imported record did not contain a valid e-mail address for a new subscriber.

If the autoresponder you chose has an immediate message, it will be sent out the next time the automated cron task runs. It cannot be sent now as there is a strong possibility that your browser or the server would time out when importing long lists. The result would be that only part of your list would be imported.
