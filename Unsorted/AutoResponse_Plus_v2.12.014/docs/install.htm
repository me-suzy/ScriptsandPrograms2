Before you begin

  1. You will need a copy of the AutoResponse Plus zip file (arplus.zip).
  2. You will need unzipping software such as WinZip.
  3. You will need an FTP client such as WS_FTP or CuteFTP.
  4. If your web server allows telnet access, you will need a telnet client such as telnet.exe supplied with Windows.
  5. You need to know the path to Perl on your web server. This is usually #!/usr/bin/perl. If you are unsure, contact your host's support desk.
  6. You need to know the path to the sendmail program on your web server. This is usually something like /usr/sbin/sendmail but is different on every server. Contact your host's support desk for this.
  7. All e-mails sent to your domain need to be forwarded to your main login account. Speak to your host's support desk about setting this up.

Step 1: Unzip the AutoResponse Plus files onto your computer

Use your unzipping software to unzip the AutoResponse Plus zip file into a temporary folder on your PC. Make sure you select the option to use folder names or preserve directories when unzipping. If the unzip is successful, your temporary folder will contain these folders:

  • data
  • docs
  • help
  • images
  • mime
  • other
  • scripts
  • styles

Step 2: Setting the path to Perl

In the 'scripts' directory, there are several files that contain the path to Perl on your server. The first line in each of these files may need to be changed (using a text editor) to show the correct Perl path for your server. The files to edit are:

  • arplus.pl
  • daily.pl
  • emailcapture.pl
  • formcapture.pl
  • import.pl
  • install.pl
  • un.pl

Step 3: Setting up the 'cgi' directories

Using your FTP client, log into your web server. Assuming your CGI-BIN directory is called 'cgi-bin', create the following directories. Please note that these are case-sensitive. Also, set the CHMOD or permission of each of these directories to 0777 [drwxrwxrwx].

  • cgi-bin/arplus
  • cgi-bin/arplus/data
  • cgi-bin/arplus/attachments
  • cgi-bin/arplus/MIME (use capital letters for MIME)

Step 4: Setting up the 'non cgi' directory

In the public part of your web space (usually the directory is called 'public', 'public_html', 'html', 'htdocs' etc), create the following directory. There is no need to set any permissions. For this example, we assume your public html directory is 'htdocs'.

  • htdocs/arplus

Step 5: Editing your 'arp-paths.pl' file

Before uploading the files, you need to edit two lines in the file 'arp-paths.pl' which is located in your 'scripts' directory. Use a text editor and follow the instructions in the file.

Step 6: Transferring the files to your web server

Using your FTP client, upload the files from each of your local directories and set the permissions as shown in the table. Again, we assume that your CGI-BIN directory is 'cgi-bin' and your public html directory is 'htdocs'.

Local Computer Directory Web Server Directory Transfer Mode File Permissions
help html/arplus ASCII Not Applicable
styles html/arplus ASCII Not Applicable
images html/arplus BINARY Not Applicable
scripts cgi-bin/arplus ASCII 0755 [drwxr-xr-x]
data cgi-bin/arplus/data ASCII 0777 [drwxrwxrwx]
mime cgi-bin/arplus/MIME ASCII 0755 [drwxr-xr-x]

Step 7: Time to run the installation program

To make things easy, we've created a program that will install your databases and create other files needed to run the system.

If you receive any error messages when you run the installer program, go back and double check everything.

  • Step 7(a) : Starting the installer program

Using your web browser, run the installer program by entering the following address:

http://www.yourdomain.com/cgi-bin/arplus/install.pl

(Obviously substitute yourdomain.com with your own domain name)

  • Step 7(b) : Starting the installation procedure

When it is run, the installer program will check certain aspects of the directories you have created. If you see any error messages, go back over the directory creation and file upload procedures then try running the installer again.

If there are no error messages, click the Install button to begin the installation procedure.

In a few seconds, the results of the installation procedure will be shown. If you see any error messages, go back over the installation instructions and double check everything before trying again.

If no errors are reported, you can test that AutoResponse Plus runs by clicking the link at the bottom of the page.

  • Step 7(c) : Delete the installer program

IMPORTANT NOTE: When the installer has run without any errors and AutoResponse Plus runs OK, you need to delete the file 'install.pl' from your 'cgi-bin/arplus' directory. If you don't, anyone can run the installer and delete all your AutoResponse Plus data.

Step 8: Configuring AutoResponse Plus

When you run AutoResponse Plus for the first time you will see a red message bar telling you that the system setup is not yet complete. Enter owner for the owner name and 12345 for the password on the login screen.

  • Step 8(a) : 5 important settings

When you have logged in, click the Setup option on the menu bar. Complete the five entries shown in red and click the Save Changes button at the bottom. AutoResponse Plus will not run correctly until these five items have been entered correctly.

The five entries are:

  • your domain name (eg yourdomain.com)
  • the URL to your AutoResponse Plus scripts directory (eg http://www.yourdomain.com/cgi-bin/arplus)
  • the path to your sendmail program
  • the e-mail address of your support desk (eg support@yourdomain.com)
  • the system e-mail address that will be used to send your subscribers confirmation messages (eg admin@autoresponseplus.com)

When you have saved these changes, the red message bar will vanish.

  • Step 8(b) : Testing the AutoResponse Plus e-mail engine

At this point, you can test if the AutoResponse Plus e-mail engine is working correctly. Click on Setup on the menu bar then click on E-Mail Send Test on the sub menu bar. Follow the instructions on the screen carefully. If you get an Internal Server Error while running this test, check these items:

  • Did you create the MIME directory in the correct place, upload the 'Lite.pm' file and set all permissions as instructed?
  • When you created the MIM directory, did you type MIME in capital letters? You must!
  • Did you enter the correct path to your sendmail program? (It's usually something like /usr/sbin/sendmail)

Step 9: Setting up .forward

All e-mails sent to your domain need to be checked to see if they are directed to one of your autoresponders. This is done using the UNIX .forward system.

  • Step 9(a) : Editing the example .forward file

The 'other' directory you created when unzipping contains an example .forward file called 'forward'. Using a text editor, edit this file.

  • Replace 'loginname' with the login name you use to access your web server.
  • Replace '/usr/bin/perl' with the correct path to Perl for your server.
  • Replace '/path/to/your/cgi-bin/arplus/emailcapture.pl' with the absolute path to 'emailcapture.pl' on your web server.

    IMPORTANT NOTE: The path to 'emailcapture.pl' you enter is from **your** domain's root and not your **host's** server root. Sometimes these are the same, sometimes they are not!

  • Step 9(b) : Uploading and renaming the .forward file

Check that your web server does not already have a .forward file in your main login directory. Note that this may not appear in your FTP client. You may need to login through telnet and use the ll (lower case LL) to see it.

Using your FTP client in ASCII transfer mode, upload the edited 'forward' file to the main login directory of your web server. Note that this file is called 'forward' when you upload it but must be renamed '.forward' after upload. This is a Windows limitation.

  • Step 9(c) : Testing the .forward file

To test that the .forward file is working correctly, click on Setup on the menu bar then click on E-Mail Capture Test on the sub menu bar. Follow the instructions on the screen carefully.

If you have problems...

Some hosts do not allow Perl scripts to be run from within the .forward file. If this is the case, you have some choices:

  1. Ask your host to remove this restriction. It is a UNIX/Linux configuration issue.
  2. Use the file .procmailrc instead. Edit the example procmailrc file in the 'other' directory on your PC, following the instructions in the file. Upload the file using ASCII to your main login directory and rename it .procmailrc
  3. Check out the Troubleshooting forum at http://www.autoresponseplus.com/forums for some more solutions to .forward problems.

The procedure for testing .forward (explained above) can also be used to test that .procmailrc is working.

It would be unusual for your server not to be able to use at least one of these methods.

Step 10 : Setting up the cron job

Once per day, AutoResponse Plus needs to send out any due follow-up messages. At this time it will also send out any one-off messages you have queued. This process is managed automatically by a cron job running on your web server. Some hosting services provide a web based interface for creating and maintaining cron jobs. If this is the case, follow the instructions provided by your host.

More often however, you will need to set up the cron task manually through telnet. Here is a detailed explanation of the procedure.

You can check if your system is already running other cron jobs from the telnet prompt. Login to your web server using your telnet client and enter the command crontab -l [lower case L]. If cron jobs are already running, you will need to add what follows to the end of your existing cron file rather than creating a new one.

  • Step 10(a) : Creating your cron file

This contains a single line description of the automated task - what has to be run and when. The 'other' directory on your PC contains an example cron file called 'arpcron' which you can edit and use. An example is shown below.

30 23 * * * /usr/bin/perl /path/to/your/cgi-bin/arplus/daily.pl

IMPORTANT NOTE: The path to 'daily.pl' you enter is from your **host's** server root and not **your** domain's root directory. Sometimes these are the same. It would be normal for your domain or login name to appear somewhere in this path.

The 30 23 * * * in the example tells UNIX that the job should be run every night at 23:30 hours (11.30pm). This can be set to whatever time you choose. If you change 30 23 to * *, the cron taks will run once per minute. Although you would never normally use this, it will allow you to test that the cron task is working -- see step 10(c) below.

If your server is already running a cron job, you will need to add your command line to the existing cron file. You can check if your server is already running any cron jobs by typing crontab -l (lower case L) at the telnet prompt.

Assuming your server is not already running cron jobs, upload your cron file using ASCII mode to a location on your web server that does not have public access (like your login root directory or a sub-directory of it).

  • Step 10(b) : Running crontab to set up the cron job

    Login to your server through telnet and navigate to the directory where you just stored 'arpcron'. Assuming your cron file is called 'arpcron' as in our example, type the command crontab arpcron and hit Enter. You should simply get the UNIX prompt back.

The command crontab -l (lower case L) will show you that the cron job is active and the command crontab -r will cancel the cron job, should this ever be necessary.

  • Step 10(c) : Running a test cron task

    As described above, if you set the cron task's time to * * * * *, it will run once per minute. When you install AutoResponse Plus, the install program creates a test autoresponder called test. It has an immediate message plus two follow-ups that are scheduled to be sent out each time the cron task runs.

So set the cron task to run once per minute (remember to run crontab arpcron) and then subscribe yourself to the autoresponder called test by sending a blank e-mail to test@yourdomain.com. You will receive the first message immediately. One minute later you will receive the second message and a minute after that you will receive the third.

This tells you that the cron task is working correctly. You can then safely set the time back to 30 23 in the cron file. Remember you need to run crontab arpcron to activate the changes you made to the file.

Step 11 : Running AutoResponse Plus

Congratulations - installation is complete!

At any time, you can reach the AutoResponse Plus login screen in your browser by going to the URL:

http://www.yourdomain.com/cgi-bin/arplus/arplus.pl

(Obviously substitute yourdomain.com with your own domain name)

It would be a good idea to bookmark this address.

The default 'Owner Name' is owner and the default 'Password' is 12345. When you have logged in to the system, click on Your Profile on the menu bar. Here you can enter a new owner name and password. We recommend you do this immediately after installation.
