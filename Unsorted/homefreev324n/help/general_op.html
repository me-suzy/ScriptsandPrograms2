General Operations
Posted on: February 28, 2000, 1:33 a.m.

the General Home Free operations section allows you to configure the main appearance and settings for your Home Free script.These variables should be setup before you start giving out accounts.

To edit these variables, simply select "General Operations" from the admin.cgi image map menu.

Each of the options allows you to change and configure your entire Home Free script, all onlineand without having to download or edit any files.You can edit and add any options you would like at a later time.

  • Amount of space to give: The amount of space alloted to each account in kilobytes. This option should not be left blank

  • Number of possible folders per account: This limit the amount of folders an account can create. A smaller number of folders may make policing account easier. To allow an unlimited number of folders, leave the text box blank.

  • Number of folders deep to allow in an account: is also very useful as it allows you to limit the total number of folder depth. This allows you to stop any user from having more then the set number of sub directories and makes it much easier to browse through a user's account then letting them have unlimited sub folders. Leave the text box blank to allow unlimited deapth in folders

  • Total number of files to allow per account: Will only allow X number of files to be uploaded or created in a persons account. Leave blank to allow unlimited number of files.(even though we say unlimited files, the user will be bound by their allotted space requirements)

  • Max kb of file that can be uploaded: The maximum size in kilobytes that will be allowed for any file upload. Leave blank to turn off max file size check.

  • Show terms and conditions on signup page: This option turns the signup conditions on or off, on the new.cgi signup page. New user must also check an "I agree" checkbox before their application is proccessed. These conditions can be edited along with your other headers and footer options. For the most part, you will want to leave this option on so that you can post your signup terms for users to agree to on the new.cgi signup form.

  • Show Solution Scripts footer on cgi generated pages: Show your love and affection for Solution Scripts, this places a small Solution Scripts credit on the bottom of every cgi generated page (new.cgi and file manger pages)This is not required by any means, although nice ;)

  • If html page has the frameset tag, put no headers or footers: This when enabled will stop Home Free from adding headers or footers to pages with frameset tags. This option will not ban headers or footers for pagesin a frame, but simply from the actual frame control page. Most sites enable this option as without it, frames will receive the header/footer code and most likely will not function correctly.

  • Allow one email address to signup for multiple accounts: When enabled allows anyone to signup for an unlimited number of accounts.This can be good if you are not worried about people signing up for many accounts and using all your disk space but should be turned off if you feel that you want to keep each user to one account only. (Also note: users with large numbers of account may want to have this feature "on" as then Home Free does not have to scroll through each member onthe sign up proccess looking for a dup email, thus making signup much faster)

  • Receive email upon a new account signup: Will send an email to the admin email address on recordwhenever a new account is added to the site. This is a very important option if you wish to keep up with account additions or if you are using the Admin Enable account password option but is not recommended for sites getting many daily signups.

  • Log IP Addresses of users: A much asked for feature, this logs accounts at the intial signup, and the last time they logged into their account.To view the ip log for an account, you must view the account itself, from the main admin screen or the account browser

  • How new accounts are set up: option allows you to select the method in which new users receive their password and/
