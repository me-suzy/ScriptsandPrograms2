Editing Categories
Posted on: March 24, 2000, 1:22 a.m.

Editing existing categories.

Once you have enabled and setup categories on your Home Free script. you may find that you want to edit or delete a category at a later time. In order to do this, you will need to use the Edit/Delete category option in admin.cgi. This option can be found by logging in to your admin.cgi script and either going to the categories and clicking the Edit/Delete category button or simply clicking the button on the main admin page (as seen below).

After clicking this button, you will be taken to a page that lists each and every one of your current categories. Each category will have an option to delete the category and show you your current settings to edit the categories.

You may edit as many or as few of the settings fields as you wish. You also can edit the settings for more then one category at a time. The options for editing a category are the same as for adding one with the exception of the delete option.
  • The "Delete" option is the only option not seen in the add category page. This option allows you to entirely remove a category from your Home Free script completely and instantly. Once a category is deleted, it can not be restored, just recreated.
  • The first option "Category Name (what users will see): " is where you should enter the title name for the category. This name will be how the category is referenced to throughout the site and should explain the category purpose briefly.
  • The next option, "Category Name (what will be added to url): " reflects what will be added to the user's url for this category. For instance, if your base home free member's directory was www.solutionscripts.com/homefree and you typed in solution for the category url, then users in this category would have the url www.solutionscripts.com/homefree/solution/account. You must enter data into this field or the category will not work. Spaces should not be used unless your server permits them in directory names.
  • The option, "Category Description:" can be a detailed or short description of what the category is for and what sites belong in it. This description will appear on the new.cgi for users to see when they choose what category to signup for.
  • The options "Category Header for html: and Category Footer for html:" should reflect the desired header and footer files to place on member's pages in this category. If you have yet to make a header/footer file for this category, you can always ad done in later.
  • The options "Category Header for manager.cgi: and Category Footer for manager.cgi:" also refer to the unique headers and footers used for that category and for view when a user logs into the account manager. These files can be added/modified at a later time and once again do not have to be used.
  • The final option, "Number of possible accounts:" is a place for you to enter how many accounts to allow per category. This can be any number you wish from one to a billion. By using this feature you can limit the total number of users on your system or just monitor how many accounts are in each category simply and efficiently.
