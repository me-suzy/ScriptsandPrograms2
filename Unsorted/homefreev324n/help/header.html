Headers and Footers
Posted on: February 28, 2000, 1:38 a.m.

The Header / Footer option allows an admin to control the html headers and footers for each category, for the new.cgi and members.cgi page, the signup terms, the code to print on new signing up sites, emails and more.

To edit a header/footer or other configuration file, simply click the "Headers/Footers" portion of the left image map menu. You will be shown a screen with links and descriptions for the files that can be configured. Files that can be configured are as follows:
  • header.txt The text above all functions in the file manager
  • footer.txt The text below all functions in the file manager
  • header_html.txt The text add to the top of all members html pages
  • footer_html.txt The text used as a footer to all members html pages
  • index.txt html for the index.html page create upon signup
  • welcome_html.txt If you have it so people can choose password
  • welcome.txt Text added to the welcome email containing members password
  • rules.txt Terms and conditions file, make users accept this before they are given an account Besides the general headers and footers, you can also edit and create custom headers and footers for each category. These files will appear on user pages if they are in the category. To do so, simply use the features below:

    • Edit allows you to edit a already created header or footer file and to add/remove/edit the html code in it.

    • Create New allows you to create a new header/footer for a specific category.

    • Delete allows you to delete a specific header/footer file.

    Once you have added a header/footer file, you must go to the edit category page and select that header/footer for the category you want it used in. After editing any header/footer file that will appear on user's pages, use the admin.cgi Update Headers/Footer features to make the new changes appear.
