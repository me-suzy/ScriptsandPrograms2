Installation Instructions

Thank you for purchasing Hotlink Protector!

Follow these simple instructions to install the script, if you require assistance or need us to install for you, contact us at help@hotlinkprotector.com

Customization
First, open up the .htaccess file with notepad. You will notice lines like
RewriteRule ^(.*)\.jpg$ r.php?a=$1.jpg [L]
To protect a file type (that is not already included in the file), insert the line
RewriteRule ^(.*)\.XXX$ r.php?a=$1.XXX [L]
where XXX is the file type you would like to protect from hotlinking. To remove a filetype, simply delete the line containing the file type from the file.
Upload the file to your server in the directory you would like to protect. To protect all directory trees, place the htaccess file in the root directory.

Next, open up r.php in notepad. There are several variables that you may need to set. $domain contains "safe" domains - users downloading from these domains will be able to download their requested file. Make sure your domain is added to the list. To add a safe domain, use the syntax $domain[X] = "domain.com"; on each line. X refers to an integer. For example, to add a third domain to the list, use:
$domain[0] = "domain1.com";
$domain[1] = "domain2.com";
$domain[2] = "domain3.com";

$logging is the option on whether to log statistics or not. Set this value to 1 to enable logging, 0 to disable it.
$password is the password needed to view the statistics. Set this to whatever you like. MySQL is required for statistics. $mini_log saves the data to a text file. Generally, if you need to reduce server load turn this on - normally this should be off.

$sessions is the option on whether to enable sessions or not. Sessions refers to the algorithm that makes sure that users must visit your site in order to download a file. This is used to allow people from downloading from download accellerators, media player, etc without allowing empty referrers to download your content. Once a user visits your page, they may download files off your server in any way they like for a timelimit set in $session_timeout. This feature requires MySQL.

$visitonly refers to whether users can download files ONLY if they have visited your website. 1 to enable this, 0 to disable it. We have coded it so you can specify which filetypes to incorporate $visitonly for. See below to $v documentation for help on how to work with this. MySQL and sessions are required for this to work.

$redirect is the option on whether you would like to redirect users to another file if they are hotlinking. Set this to 1 to enable it, 0 to disable. Additional infomation on how to edit the files sent to hotlinkers is below.

$send_headers is used if you want the user to be redirected to a similar filetype. It helps the browser determine what is being sent to them. If youre redirecting to a website or something, turn this off.

$country_redirect allows you to redirect users from a specific country to another page. $redirect_url is the url to redirect them. When installing the script, the country table may take a while to be created, since it contains 15000+ records. Download the latest database from http://www.hotlinkprotector.com/ip-to-country.csv, and upload it to your webserver. To set the countries, edit the $cr variables. Youll need the exact country name. To get them, simply view the csv file with excel.

$addiptosite automatically adds users ips to the database if they visit one of your webpages. Edit $pagetypes to set the extensions of your website. Be sure to edit your .htaccess file and add the filetypes to there so the script can catch requests to those filetypes.

Additional headers are not neccessary, but help a lot in helping the browser determine what type of file is being sent, as well as its filesize and type. They follow the syntax $m[a][b] where a is the group, and b is the data. When b is 0, it refers to the filetype for which the following headers are sent for. When b > 0, the script will send the headers from 1 to infinity. Most common filetypes are already accounted for, and you will likely not have to edit this. $ma[x] are headers sent for ALL filetypes. Headers are very powerful. You can add cache information in headers or redirect to external servers.

$r[x] refers to the files to send to users that hotlink files. x is the group number for the files. If you want to redirect users that hotlink a gif from you to another image, look up gif in the table of $m[a][0], and youll find it in the line $m[5][0] = "gif";. Insert the line $r[5] = "file.gif"; to redirect hotlinkers that request *.gif to file.gif.

$v[x] follows the same principle as $r[x]. x again is the group number of the files. If you want to enable make sure visitors can only download a file if they have visited your page, set $v[x] = "1";. This is best used for large files, such as movies only. This will only work if a users IP has been added to the MySQL table. The IP is automatically added to legit users that download a filetype where $v[x] is not 1, such as an image on your website that you have not set the filetype to 1.

Installation
After configuring the r.php, upload it to your webserver in the directory of htaccess. Open up the file r.php in a webbrowser to install the script (specifically the mysql tables). Visit r.php?p=PASSWORD (the password you set in the file) to view statistics and to make sure that the script was installed correctly. That should be it, your script is all setup!

Commands for r.php?
p=PASSWORD: This loads up the stats log
p=PASSWORD&c=fluship: This clears the ip table. Used for debugging
p=PASSWORD&c=flushall: This clears all tables. Used for debugging
p=PASSWORD&a=FILENAME&ref=REFERRER: This manually sets a specific filename, and referrer to the script. Used for debugging
