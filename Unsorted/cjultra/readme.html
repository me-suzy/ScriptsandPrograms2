GUIDE FOR CJULTRA v2.1

Requirements:

-Your server has to support PHP4 and Mysql.
-You need to have a Mysql database in your server and username/password for it.(these are sometimes same as your ftp login but still make sure). Ask your hosting to set up one for you.
- You need to have basic webmaster skills including: how to edit the html source code, how to ftp and upload files in ASCII and Binary modes, how to chmod files through your ftp program...

Installation of CjUltra-Free:

If you already have Cjultra installed, skip this and read the next section. 

You need to understand and follow every step or cjultra is probably not gonna be properly installed.

-Upload all the files to your root directory (recommended) or to another sub directory.
-Make sure you upload php files in ASCII mode and the gif files in Binary.
-Create a subfolder where your script is, call it 'cjstats' and chmod it to 777
-Chmod these files to 777: iplog.txt, topheader.txt, topfooter.txt, toplines.txt, common.php
-Your main page needs to end with .shtml or .php. The main page is where the trades are gonna send the hits to. Most people use index.shtml

-For SSI (.shtml) Add this line in the header part of the main page html code : 
<!--#include file="in.php"-->

-For PHP (.php) Add this line in the header part of the main page html code : 
include("in.php");

-If you installed the script to a subfolder of your main page, use this instead of the one above: 

for SSI:
<!--#include file="foldername/in.php"-->

for PHP:
$cju_dr = getcwd();
chdir($cju_dr . "/foldername");
include("in.php");
chdir($cju_dr);

 

-Open setup.php with your browser and fill out the form with your info.
-Edit rules.txt This will be shown at the top of the webmaster sign-up page.

-If you get errors make sure the mysql info is correct and you did the chmoding correctly.

Does my script work?

Go to your admin, if you can login without any problems, it is properly installed.
Also go to your main page with your browser and look into your html code. You should see some javascript code at the place where you put the in.php , which may look like this:

<script language="JavaScript">
<!--
document.cookie='from=noref; expires=Thursday, 20-Feb-03 21:34:12 GMT;';
//-->
</script>

 When you open your site with your browser, if you see <!--#include file="in.php"--> in your html code, that means SSI is not running. If you dont turn on SSI , the script wont work. Ask your host to turn it on, if you couldnt see the javascript code.

To Upgrade from previous versions:

- Upload all files except common.php and uninstall.php . If you accidently upload common.php , you will need to run setup again.
- (Unless it already exists) Create a subfolder where your script is, call it 'cjstats' and chmod it to 777
- Chmod these files to 777: iplog.txt, topheader.txt, topfooter.txt, toplines.txt, common.php
 

 

Some Terms:

R.In : Number of Raw hits incoming from that trade or domain
U.In : Number of Unique hits incoming from that trade or domain
U.Out : Number of outgoing hits to that trade
Clicks : Number of clicks generated by the incoming hits from that trade.
Prod : Productivity of the trade. Calculated by: Clicks / R.In x 100
Daily : The stats since the last reset.
Hourly Force : Number of minimum hits to send to that trade every hour (if there is enough available hits of course).
Actual Ratio : The ratio of hits you send back to that trade.
noref : It stands for the surfers who opened your page directly or their browsers didnt carry the referring url info.
exout : How many hits you send to your exout url. If someone visits all your trades, he will go there.
nocookie : Counts the clicks from the surfers who has cookies or javascript disabled.

The table on the top shows the totals for the site. The totals at the bottom of the table shows the total of the stats that appear on the table. Thats why they usually differ.

How to use the script:

How the links look like:

You are gonna be sending hits to your trades through out.php . Otherwise they can not be counted.

example urls:

out.php

out.php?link=abc

out.php?url=http://www.somehost.com/gallery.html&link=gal&s=70

out.php?perm=tradedomain.com&link=trade


As you can see, there are parameters like: link= , url=, s= , perm=, first=

The order of them is not important, you just need to seperate them with an ampersand (&)

Read below to see the usage of these parameters

Links Stats:

This feature is used to track which links are clicked and the number of the clicks.

usage:

out.php?link=blah

Go to the "Link Report" section and see how many times the links with those labels are clicked on.

Permanent Linking:

With this feature you can put a permanent link on your page to a trade. Whenever someone clicks on that link, he will always go to that trade and those hits will be counted by the script.

examples:

out.php?perm=domain.com

out.php?link=dom&perm=domain.com

out.php?perm=domain.com&link=blabla

Note: the trade has to be listed on your trades list and you need to put the exact same domain name after perm= , or the hit will go to a random trade.

Skimming (Sending hits to galleries):

Most important feature for tgps and for cj/cj2 sites who wanna put some content on their page.

example:

out.php?url=http://www.somehost.com/gallery.html&link=gal&s=70

When a surfer clicks on this link, he will go to the gallery url (as stated in 'url=' parameter) by 70% chance, and he will go to one of your trades by %30 chance. As i said before the order of parameters 'url=' , 'link=' and 's=' are not important, just seperate them by an ampersand (&). s= and link= are optional. If s= is not in the url, the default skimming is 50%.

Note: If the url contains characters like "?" or "&" or "=" , then you need to encode the url first and then put it after "url=" in your links. To encode a url you can use this form : http://www.cjultra.com/urlencoder.php

First X Clicks Skimming:

This is a small but very usefull feature added to the skimming feature. It helps increasing your productivity a lot on most cases.

example:

out.php?url=http://www.somehost.com/gallery.html&link=gal&s=70&first=1

When a surfer clicks on this link, and if this is his first click on one of your out.php links on your site, he will go to the url by 100% chance. On further clicks he will go by 70% of chance.

(added in v2.1) If you put a number greater than 1, let's say 2 (first=2), the first 2 clicks will go to the url.


Admin page:

The top table shows general statistics for the site. It is not only the total of your trades, its the total of your whole site. Daily hits are the amount of hits since midnight.

Adding trades:

Use webmaster.php or the "Add Trade" option in your admin ; if you want to show additional rules or other stuff on top of the sign-up page, edit rules.txt.

Referrer Stats:

These stats show you which domain sent how many hits. You can also see the hourly stats for each of them.

Traffic Stats:

These stats show you the hourly stats for the site as general.

Settings:

You can change your general settings here.

Blacklist:

You can block domains or ip addresses with this feature.

 

Auto Toplist:

Instructions are located on the admin page.

New Template-based Toplist:

Toplist will update every minute. You can also manually update it by opening toplist.php on your browser window.

The script is template based, this gives you a lot of flexibility. Your template filenames have to begin with "cjutop_" and the rest is the name for the toplist file to be created. You have to create an empty file for the toplist and chmod it to 777. The script detects the files that begin with "cjutop_" and tries to create a toplist for each of them. This way you can have multiple toplists on your site.

Example:
template file name: cjutop_mytoplist.html
toplist file name: mytoplist.html (create this file and chmod to 777)

To insert the toplist into your webpage using SSI (shtml):

<!--#include file="mytoplist.html"-->

To insert the toplist into a php page:

include("mytoplist.html");

wherever you want the toplist to appear.

 


Here are the keywords you are allowed to use in your templates:

'#' stands for the rank of the site. It's a number (1,2,3...)

For a toplist based on incoming raw hits, use these keywords:

%%R_RIN#%% R.IN for the day
%%R_OUT#%% Out for the day
%%R_CLK#%% Clicks for the day

%%R_HRIN#%% R.IN for the hour
%%R_HOUT#%% Out for the hour
%%R_HCLK#%% Clicks for the hour

%%R_URL#%% Url
%%R_DES#%% Description (Site Name)
%%R_LNK#%% Script url (outputs: out.php?perm=domain.com&link=top#_domain.com)
%%R_DOM#%% Domain


For a toplist based on incoming unique hits, use these keywords:

%%U_UIN#%% U.IN for the day
%%U_OUT#%% Out for the day
%%U_CLK#%% Clicks for the day

%%U_HUIN#%% U.IN for the hour
%%U_HOUT#%% Out for the hour
%%U_HCLK#%% Clicks for the hour

%%U_URL#%% Url
%%U_DES#%% Description (Site Name)
%%U_LNK#%% Out url (out.php?perm=domain.com&link=top#_domain.com)
%%U_DOM#%% Domain


'#' stands for the rank of the site.

For example you can create a file called cjutop_mytoplist2.html and put a template like one of the above into this file. Then upload an empty file called mytoplist2.html and chmod it to 777. The script will generate the toplist in mytoplist2.html. In this example you can put the toplist on your page using this SSI tag:

 

<!--#include file="mytoplist2.html"-->



You can also create a file called cjutop_index.shtml and chmod your index.shtml to 777. (You can do this if toplist.php and all cjultra files are in the same folder as your index page). Then you can put the keywords into your page html code. You can also use the same keyword more than once in the same file. This way you can have the script directly put the toplists into your mainpage.
