Freelancers (v2.5) Setup Instructions
Skip to bottom for upgrading instructions...

Setup:
  1. If you want to accept credit card or paypal deposits, read the R3N3 Payment Gateway script instructions first (here).
  2. Unless your path to Perl is /usr/bin/perl, change it at the top of all CGI files, by opening them in a text editor like Wordpad.
  3. Open vars.cgi and follow the instructions inside to set 2 simple variables.
  4. Open cron.cgi, cron_notify.cgi and cron_freeze.cgi and specify the full path to the script near the top.
  5. Create a directory in your cgi-bin (the one you specified in vars.cgi) for this script. Such as "/cgi-bin/free".
  6. Upload all files with a .cgi extension to that directory, in ASCii mode. Upload the "html" and "data" folders with their contents too.
  7. Set the permissions of all CGI files to CHMOD 755.
  8. Set the permissions of the "html" ("/cgi-bin/free/html") and "data" ("/cgi-bin/free/data") directories to CHMOD 777, do the same with all files inside those 2 sub-directories.
  9. Create the following sub-directories inside "/cgi-bin/free": "freelancers", "buyers", "projects", and "forum". Change the permissions of these directories to CHMOD 777.
  10. Create a directory outside your cgi-bin for graphics (such as "/graphics"). Change the permission of this directory to CHMOD 777. Then upload all graphics into that directory, in Binary mode
  11. If you want to allow people to attach files to a project, create a directory outside your cgi-bin for them (such as "/files"). Change the permission of this directory to CHMOD 777
  12. If you want project pages to be saved as HTML files (takes up less system resources), create a directory outside your cgi-bin for them (such as "/projects"). Change the permission of this directory to CHMOD 777
  13. Now run admin.cgi in your browser, and you will finish setting up the script.
  14. Setup Cron to run the file cron.cgi once a day. Run it under root, if you have access to it. If your server doesn't have Cron, either do it manually every day, or check out our free www.smarterscripts.com/cron Fake Cron" CGI. For best results you should only run it once a day.

How to use:

  • You can change the settings of the script and edit the html templates anytime you want from the admin, just click on the "customize" button.
  • You can find examples in /html/header.html on how to link to parts of the script (signing up, creating projects, etc...)
  • If you want to use your own graphics, just make sure the filenames are the same, and upload them to the directory you put graphics in for this script.
  • If you want to clean your database of expired projects more than once a day, you can run cron_freeze.cgi with Cron. Run it every few minutes or every hour.
  • If you have many freelancers, you should run cron_notify.cgi with Cron every few minutes. Make sure to turn this feature on in the Misc section of admin settings.
  • You can display information using SSI or Javascript tags.

    SSI Example: <!--#include virtual="/path/to/cgi/d_top.cgi" -->

    Javascript Example: <script src="http://www.mysite.com/d_top.cgi?show=5&java=1"></script>

    Note: You must add &java=1 to the URL when using the Javascript tag.

    • d_top.cgi displays freelancers in order of highest rating to lowest. Give a value to the variable "show" to specify how many freelancers should be displayed. If you want to show "special" users first, put s=1.
      Example: <!--#include virtual="/path/to/cgi/d_top.cgi?show=10&s=1" -->
    • d_count.cgi displays the total number of freelancers, buyers, or open projects. Add b=1 to get the buyer count, f=1 for freelancers, and p=1 for the number of projects.
      Example: <!--#include virtual="/path/to/cgi/d_count.cgi?b=1" -->
    • d_projects.cgi displays projects, from newest to oldest. Give a value to the variable "show" to specify how many projects should be displayed. To display only featured projects, add f=1
      Example: <!--#include virtual="/path/to/cgi/d_projects.cgi?show=20&f=1" -->
    • d_topprojects.cgi displays projects, from the highest average bid to the lowest. Give a value to the variable "show" to specify how many projects should be displayed.
      Example: <!--#include virtual="/path/to/cgi/d_topprojects.cgi?show=20" -->
    • If you want make the title of projects displayed by d_projects.cgi or d_topprojects.cgi look nicer, add text=1. This will convert the title to lowercase letters, except for the first letter of each word, which will be made uppercase.
      Example: <!--#include virtual="/path/to/cgi/d_projects.cgi?show=20&text=1" -->
  • Affiliates can display your projects with the d_projects.cgi file too. They just need to add &r=USERNAME to the URL (and replace USERNAME with their username).
    Example: <script src="http://www.yourdomain.com/d_projects.cgi?show=20&java=1&r=USERNAME"></script>

  • Affiliates can get a database text-only print out of the projects by adding db=1: http://www.yourdomain.com/d_projects.cgi?show=20&r=USERNAME&db=1. This prints out the projects 1 per line, with each data field seperated by a tab space. The following project data is printed, in order: title, url, number of bids, start time (in epoch seconds), end time, categories (seperated by pipelines: | ), and the word "featured" if the project is featured. It's not a problem if you don't understand this, any affiliate who chooses to use this feature will be a programmer and will understand it.

R3N3 Payment Gateway:

To accept credit card and paypal payments with Freelancers, you will need to have our Payment Gateway script installed on your server. This script is free to customers and comes with your download of Freelancers, in the same Zip file. If you already installed it once, you can use that one for Freelancers too. Otherwise, open the Zip file with the filename "PaymentGateway" and read its setup_instructions.html file to install and setup that script. Install it before you continue to setup Freelancers.

Upgrade Instructions:

Upgrading from version 2.1: (previously NULLED BY WST)
  1. Replace/Upload ALL files in the main folder that have a .cgi extension. Make sure the path to perl is correct at the top of each. Reset the two variables required in vars.cgi and the one in the Cron files (cron.cgi, cron_notify.cgi and cron_freeze.cgi). Set all CGI file permissions to CHMOD 755.
  2. If you want to save project pages as HTML files, create a directory outside of your cgi-bin (such as "/projects") and set the permissions to CHMOD 777.
  3. There are a few new variables to define in the "paths" and "new" section of the admin setup area.

Upgrading from version 2.0:

  1. Replace/Upload ALL files in the main folder that have a .cgi extension. Make sure the path to perl is correct at the top of each. Reset the two variables required in vars.cgi and the one in the Cron files (cron.cgi, cron_notify.cgi and cron_freeze.cgi). Set all CGI file permissions to CHMOD 755.
  2. Upload the "graphics/arrow.gif" image into the directory your other Freelancers graphics are in.
  3. If you want to clean your database of expired projects more than once a day, you can run cron_freeze.cgi with Cron. Run it every few minutes or every hour.
  4. If you have many freelancers, you should run cron_notify.cgi with Cron every few minutes. This sends out new project notifications e-mails. Make sure to turn this feature on in the Misc section of admin settings.

Upgrading from versions 1.0-1.1:

  1. If you want to accept credit card or paypal deposits, read the R3N3 Payment Gateway script instructions first (above).
  2. Replace/Upload ALL files in the main folder that have a .cgi extension. Make sure the path to perl is correct at the top of each. Reset the two variables required in vars.cgi and the one in cron.cgi too. Set all CGI file permissions to CHMOD 755.
  3. Upload ONLY the new file vars_new.cgi into the "data" subdirectory, leave the other files from that folder alone. Set the permissions of vars_new.cgi to CHMOD 777.
  4. Upload the "graphics/flag.gif" and "graphics/arrow.gif" images into the directory your other Freelancers graphics are in.
  5. Login to the admin, go to the setup area (click on the "Custmize Script" button), and define the new variables. There is one new variable in the "Admin/Site Info" group called "from_email", and a whole new group labeled "New". Also, the "Paths" group has changed, make sure to look at it, especially if you are using the R3N3 Payment Gateway script.
  6. The account management template ("html/manage.html") has a new tag: [escrow]. Insert that tag in the template if you want to allow members to pay with escrow accounts.
