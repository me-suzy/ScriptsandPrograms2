Sponsors

 More info of how the sponsored system works may be found here.

An existing sponsor can log in at 's_user.php'. The login page is based on template s_login.html. This template has several variables, these are used only if someone already tried to log in (filled in the form and sent it out) but the system rejected it for some reason (probably wrong username or password). The following variables are available:

#%info%#

Description of the reason why the form was rejected

#%userid%#

Entered Username

#%userpass%#

Entered Password

If a member forgot username or password, he/she will go to 's_user.php?action=remind' for a password reminder. There the member fills in email address associated with an account and both username and password will be automatically emailed to this email. The remind form is based on template 's_remind.html'. This template has 2 variables, these are used only if someone already entered an email but the system rejected it because the entered email was not associated with any account. These variables are below:

#%info%#

Description of the reason why the form was rejected

#%email%#

Entered Email

The remind email is based on template 'email_remind.txt' where are these variables available:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%loginurl%#

URL of your s_user.php script

Once a member is logged in, he/she can see a page based on template 's_home.html'. There are these variables available:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%w1%#, #%h1%#
#%w2%#, #%h2%#
#%w3%#, #%h3%#

Banner dimensions (width, height) used in 3 sizes of your exchange system.

On this page the user can select from several actions to do. Below is a description of each of these actions and list of variables in used templates.

Edit details
By selecting this action a member gets a form to edit user details.  This form is based on template 's_edit.html'. These variables are available:

#%info%#

If the form was just send out, it informs the user that data are saved or that some error occurred.

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%name%#

Name of the member

#%email%#

Email of the member

If you have checked that you want to get an email when a member changed his/her details, you now get an email based on template 'email_admin_s_change.txt'. These variables may be used:

#%adminfile%#

URL of your admin interface

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%name%#

Name of the member

#%email%#

Email of the member

Show orders
By this function your sponsor gets a list of his/her orders. There is used template 's_orders.html' to show this list. Used variables:

#%userid%#

Username of the sponsor

#%userpass%#

Password of the sponsor

#%orders%#

List of orders, each line of this list is based on template 's_order.txt' (see below)

To show each order in the list above is used template 's_order.txt'. These variables may be used:

#%number%#

Order number

#%price%#

Price of the order

#%bonus%#

Value of a bonus which contains the ordered package (in %)

#%value%#

Total value of the order (price + bonus)

#%status%#

Paid or Not paid

#%date%#

Order date

#%paylink%#

A button or link going to your payment page (only if the order is not paid yet)

Order/manage funds
This option allows your sponsors to add clicks or impressions to their accounts, move these and also to add (purchase) new funds to their accounts. There is used template 's_funds.html' for this page. Used variables:

#%info%#

If the user just made any action (ordered clicks or impressions etc.), it informs her/his about what happened.

#%userid%#

Username of the sponsor

#%userpass%#

Password of the sponsor

#%w1%# #%h1%#
#%w2%# #%h2%#
#%w3%# #%h3%#

Widths and heights of banners in sizes 1, 2 and 3 of your system.

#%pr_clicks1%#
#%pr_clicks2%#
#%pr_clicks3%#

Actual price for 100 clicks in sizes 1, 2 and 3. You must set up these prices in your Configuration.

#%pr_imp1%#
#%pr_imp2%#
#%pr_imp3%#

Actual price for 1000 impressions in sizes 1, 2 and 3. You must set up these prices in your Configuration.

#%clicks1%#
#%clicks2%#
#%clicks3%#

Currently available, not used clicks which this user has in sizes 1, 2 and 3. The user can cancel these clicks and exchange them back for funds. If that happens, these funds are given back to the user's main account, so he/she can use those for another purposes (for example to order impressions). Please note that there is always used actual price (the price for 100 clicks entered in your Configuration) for this action, not depending of a price which was valid in the time of the initial order.

#%imp1%#
#%imp2%#
#%imp3%#

Currently available, not used impressions which this user has in sizes 1, 2 and 3. The user can cancel these impressions and exchange them back for funds. If that happens, these funds are given back to the user's main account, so he/she can use those for another purposes (for example to order clicks). Please note that there is always used actual price (the price for 1000 impressions entered in your Configuration) for this action, not depending of a price which was valid in the time of the initial order.

#%balance%#

Currently available balance on user's main account. User can use these funds to add clicks and/or impressions to his/her accounts.

#%packages%#

List of sponsor packages which you have. User can select one of them to add a new funds to his/her main account. You can add/edit these packages in Admin Area.

Once an user ordered new funds, he/she gets page which is based on template 's_ordered.html'. On this page your user gets a link or button going to your payment processor to pay for the ordered package. Used variables:

#%userid%#

Username of the sponsor

#%userpass%#

Password of the sponsor

#%descr%#

Description of the just ordered package

#%payhtml%#

A link or button going to your payment page

Once an order is placed, you (admin) get an email based on template 'email_admin_s_order.txt'. Used variables:

#%adminfile%#

URL of your Admin Area

#%userid%#

Username of the sponsor

#%descr%#

Description of the just ordered package

#%price%#

Price of the just ordered package

#%bonus%#

Value of a bonus which contains the ordered package (in %)

#%value%#

Total value of the order (price + bonus)

Edit ads
If this is selected, your user gets a form to edit ads. This page is may be based on the following templates:
s_ad_edit.html - this template is used if you don't allow users to enter a plain HTML code and banners of your users are stored on their servers
s_ad_edit_raw.html - this template is used if you allow users to enter a plain HTML code and banners of your users are stored on their servers
s_ad_edit_upload.html - this template is used if you don't allow users to enter a plain HTML code and banners of your users are stored on your server
s_ad_edit_upload_raw.html - this template is used if you allow users to enter a plain HTML code and banners of your users are stored on your server
In these templates are available the following variables:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%size%#

Number of account which is just edited (number 1, 2 or 3)

#%info%#

It shows an info for the member when data has been saved or when any problem has occurred.

#%ienabled%#

It informs if the account is actually enabled or disabled.

#%approved%#

It informs if the account is actually approved by admin.

#%enablebutton%#

If you do allow your members enable/disable their accounts, it makes a button to enable it if the account is disabled or disable it if the account is enabled.

#%category%#

List of categories, member can select category of this account.

#%current_ad_1%#
#%current_ad_2%#
#%current_ad_3%# 

Current ads of this account.

#%current_banner_1%#
#%current_banner_2%#
#%current_banner_3%# 

Current banners of this account (only if banners are stored on your server)

#%url1%#
#%url2%#
#%url3%#

URL's which gets a surfer when clicks on link of this account.

#%alt1%#
#%alt2%#
#%alt3%# 

Alt tags of banners

#%raw1%#
#%raw2%#
#%raw3%# 

Raw HTML of ads (only available if you allow users to enter raw HTML)

Once an ad is updated, the account is disabled (if it's checked in your Configuration) and an admin must check the new info and re-enable this account. If you have selected that you want to get an email when a link is changed, you now get an email which is based on template 'email_admin_s_ad.txt'. These variables may be used:

#%adminfile%#

URL of your admin/index.php script

#%userid%#

Username of the account

#%category%#

Category of site of the account

#%size%#

Number of exchange size which has been updated (1, 2 or 3)

#%url1%#
#%url2%#
#%url3%#

URL's which gets a surfer by a click on ad of this account.

#%banner1%#
#%banner2%#
#%banner3%#

Banners of this account.

#%alt1%#
#%alt2%#
#%alt3%# 

Alt tags of banners of this account.

#%userpass%#

Password of the member

#%c_nu%#

Total number of actually available clicks

Total number of impressions received by this account

Total number of clicks received by this account

 Ratio impressions/clicks on ads of this account

#%i_free%#

Total number of free impressions which got this account

#%c_free%#

Total number of free clicks which got this account

#%hourly_graph%#

Complete table containing graph with hour-after-hour statistic

Statistic for individual banners

#%ban1%#
#%ban2%#

#%ad_i1%#
#%ad_i2%#

#%ad_ratio1%#
