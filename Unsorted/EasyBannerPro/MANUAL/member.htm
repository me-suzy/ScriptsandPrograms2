Members

An existing member can log in at 'member.php'. The login page is based on template u_login.html. This template has several variables, these are used only if someone already tried to log in (filled in the form and sent it out) but the system rejected it for some reason (probably wrong username or password). The following variables are available:

#%info%#

Description of the reason why the form was rejected

#%userid%#

Entered Username

#%userpass%#

Entered Password


If a member forgot username or password, he/she will go to 'member.php?action=remind' for a password reminder. There the member fills in email address associated with an account and both username and password will be automatically emailed to this email. The remind form is based on template 'u_remind.html'. This template has 2 variables, these are used only if someone already entered an email but the system rejected it because the entered email was not associated with any account. These variables are below:

#%info%#

Description of the reason why the form was rejected

#%email%#

Entered Email

The remind email is based on template 'email_remind.txt' where are these variables available:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%loginurl%#

URL of your member.php script

Once a member is logged in, he/she can see a page based on template 'u_home.html'. There are these variables available:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%w1%#, #%h1%#
#%w2%#, #%h2%#
#%w3%#, #%h3%#

Banner dimensions (width, height) used in 3 sizes of your exchange system.

#%move_impressions%#

If you allow users to move impressions between sizes, it shows a form field to select this option. This field is based on template 'u_move_impressions.txt'. This template doesn't have variables.

On this page the user can select from several actions to do. Below is a description of each of these actions and list of variables in used templates.

Edit ads
If this is selected, your user gets a form to edit ads. This page is may be based on the following templates:
u_ad_edit.html - this template is used if you don't allow users to enter a plain HTML code and banners of your users are stored on their servers
u_ad_edit_raw.html - this template is used if you allow users to enter a plain HTML code and banners of your users are stored on their servers
u_ad_edit_upload.html - this template is used if you don't allow users to enter a plain HTML code and banners of your users are stored on your server
u_ad_edit_upload_raw.html - this template is used if you allow users to enter a plain HTML code and banners of your users are stored on your server
In these templates are available the following variables:

#%userid%#

Username of the member

#%userpass%#

Password of the member

#%size%#

#%info%#

It shows an info for the member when data has been saved or when any problem has occurred.

#%ienabled%#

It informs if the account is actually enabled or disabled.

#%approved%#

It informs if the account is actually approved by admin.

#%enablebutton%#

If you do allow your members enable/disable their accounts, it makes a button to enable it if the account is disabled or disable it if the account is enabled.

#%category%#

List of categories, member can select category of this account.

#%czct%#

List of categories which have to be displayed on pages of this user. Users can select ANY or maximum of five single categories.

#%current_ad_1%#
#%current_ad_2%#
#%current_ad_3%# 

Current ads of this account.

#%current_banner_1%#
#%current_banner_2%#
#%current_banner_3%# 

Current banners of this account (only if banners are stored on your server)

#%url1%#
#%url2%#
#%url3%#

URL's which gets a surfer when clicks on link of this account.

#%alt1%#
#%alt2%#
#%alt3%# 

Alt tags of banners

#%raw1%#
#%raw2%#
#%raw3%# 

Raw HTML of ads (only available if you allow users to enter raw HTML)

Move impressions between sizes
This option is only available if you allow users to move impressions between ad sizes. This may be used for example if an user earns impressions in size 1 but wants to use them in size 2. If you allow it, you must also enter a ratio (in Configuration) which should be used for this conversion. Please note that while the conversion is the number of impressions always rounded down in order to prevent users to steal impressions by moving it many times between sizes.
By selecting this option your users get a page containing form which may be used to convert the impressions. This page is based on template 'u_move_impressions.html', these variables are used:

#%info%#

This is used only if some impressions have been just converted or if some error occurred to show an info what has happen.

#%move1%#
#%move2%#
#%move3%# 

The ratio which you have set to convert impressions.

#%w1%# #%h1%#
#%w2%# #%h2%#
#%w3%# #%h3%# 

Dimensions (width,height) of 3 ad sizes in your system

#%available1%#
#%available2%#
#%available3%# 

Number of currently available impressions in sizes 1, 2 and 3. This is the maximum number of impressions which can be converted.

#%userid%#

Username of the member

#%userpass%#

Password of the member
