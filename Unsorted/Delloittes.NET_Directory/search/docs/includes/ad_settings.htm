Directory Settings

The Directory Settings page allows you to control properties to determine how your directory behaves.

General Settings

Directory Name - You can enter your site / directory name within this box. This is used within your newsletters and through various sections of the directory.

Navigation Separator - This allows you to customise the character used to separate the top navigation links within the directory.

Homepage Settings

How many "Latest Addition" links to show on homepage - This allows you to control how many latest additions are displayed on the directory homepage. (This setting can be over-written if a user chooses to personalise the directory)

How many "Popular Resource" links to show on homepage - This allows you to control how many popular resources are displayed on the directory homepage. (This setting can be over-written if a user chooses to personalise the directory)

Directory Settings

Amount of sub items to show under categories - This allows you to specify how many sub categories or listings are displayed below category titles. (This setting can be over-written if a user chooses to personalise the directory)

Directory results per page - This allows you to specify how many items are displayed per page within a directory category. (This setting can be over-written if a user chooses to personalise the directory)

Search results per page - This allows you to specify how many search results are displayed per page when a user performs a search. (This setting can be over-written if a user chooses to personalise the directory)

Number of latest additions within newsletter - This allows you to specify how many latest additions are displayed within your newsletter. For example if you distribute your newsletter daily than you may only want to display the last 5 resource additions. If you distribute a monthly newsletter then you may want to display the last 50 resource submission.

Email Settings

Various features within the directory require the ability to send emails from the server. For example, if a user submits an error report or you wish to be notified of submissions. The deloittes.NET Directory is designed to work with several popular email components to ensure compatibility with your current hosting environment. The information within this section is used to determine how emails are sent from the server and also who from.

Email Component - Use this option to select which email component you would like to use to send emails. You will need to check with your host to determine which components they support.

Send email alert for resource submissions - Use this option if you would like to receive email notifications for resource submissions. If you choose not to receive submission notifications please remember to check the approve resources page regularly, as submissions will not be automatically approved.

Send email alert for review submissions - Use this option if you would like to receive email notifications for review submissions. If you choose not to receive review notifications, please remember to check the approve review page regularly, as reviews will not be automatically approved.

Send email alert for error submissions - Use this option if you would like to receive email notifications for error reports. If you choose not to receive email notifications, please remember to check the errors page regularly.

Your Email Address - This will be the return address of all emails sent from the site.

SMTP MailServer - This is the SMTP server used to send emails. If your SMTP server requires authentication please consult the emailsubs.asp file within the root of the directory application. The required SMTP authentication code is commented within the sub-routines. (CDONTS does not require an external SMTP server.)

NOTE : This page is only available to users with administration rights.
