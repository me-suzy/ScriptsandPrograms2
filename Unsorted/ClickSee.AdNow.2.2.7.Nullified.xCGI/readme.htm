2. CREATING THE DATABASE

2.1 MS Access Database Users:
Please upload the file Adnow.mdb (provided in the zip) to a server. This Microsoft Access database file contains ALL the necessary statistical and advertising data. You don't want anyone to view, have access or download it from your server.

At this point, you basically have three alternatives to ensure nobody has access to this file.

 We would recommend you to FTP the file adnow.mdb to a secure directory. Ideally it should be somewhere under the root directory that is not accessible from the Web.

 You can use Microsoft FrontPage 2000 to upload the adnow.mdb file by using the "import" function. FrontPage will ask you "Would you like to create a new database connection for this imported file?".  Choose "Yes" and let FrontPage put the file in the fpdb directory.

 If you are unable to place the file in a directory that is not accessible from the web, or use MS FrontPage to upload the file, please ask your web host or your server administrator for the secure directory.

Note to Upgrade Users: If you are upgrading from Clicksee Adnow Version 1.0 and would like to carry the advertising data to Version 2.0, please see Appendix A on how to upgrade the database from Version 1.0 to Version 2.

4. SET UP DSN_Connection.asp FILE

4.1 SETTING THE DATA SOURCE NAME

Open the file DSN_Connection.asp (located in Data_Connection folder) with any HTML/Text Editor. (Ex. NotePad) and look under the topic "SET UP DATABASE CONNECTION" you will find 4 blocks of codes: DSN MS Access, DSN MS SQL, DSNLess MS Access, and DSNLess MS SQL.

You will be editing only ONE block of code.


'******** DSN MS Access ********
'Const DSNad=""
'Const UID=""
'Const PWD=""
'*******************************

'******** DSN MS SQL ***********
'Const DSNad=""
'Const UID="your_username"
'Const PWD="your_password"
'*******************************

'***** DSNLess MS Access *******
'ConstDSNad="driver={Microsoft Access Driver (*.mdb)};DBQ=c:\yourdirectory\adnow2\csdb\adnow.mdb;"
'Const UID=""
'Const PWD=""
'*******************************

'***** DSNLess MS SQL **********
'ConstDSNad="driver={SQL Server};server=SQLservername;uid=SQLusername;pwd=SQLpassword;database=SQLdatabasename;"
'Const UID=""
'Const PWD=""
'*******************************

Notice that all the lines are commented out with a single quote (').  The ' signifies that the line of code is NOT read and processed by the server. You will have to decide which of the 4 categories your particular situation fits within, then fill in your required information, uncomment your selected block and DO NOT uncomment the other 3 blocks.

Please select ONE category that fit your situation.

4.1.1 DSN with Microsoft Access database
Use this option if you setup an ODBC system data source in the previous step.

*** Edit the block DSN MS Access as follows. ***

  • Type the Data Source Name within the quote Const DSNad. For example, Const DSNad="ClickseeAdnow2" This is the data source name that you set up in the previous step.
    NOTE: You can name it anything you want using alphabet and numbers ONLY.       
  • Remove the single quote mark in front of Const DSNad, Const UID, and Const PWD.

You will get something like this

'******** DSN MS Access ********
Const DSNad="ClickseeAdnow2"
Const UID=""
Const PWD=""
'*******************************
 

4.1.2 DSN with Microsoft SQL database
Use this option if you setup an ODBC system data source in the previous step.

*** Edit the the block DSN MS SQL as follows. ***

  • Type in the Data Source Name within the quote Const DSNad. For example, Const DSNad="ClickseeAdnow2" 
    NOTE: You can name it anything you want using alphabet and numbers ONLY.       
  • Type in a user name and a password that you use to access your SQL     database in "Const UID" field and "Const PWD" field   
  • Remove the single quote mark in front of Const DSNad, Const UID, and Const PWD.

You will get something like this

Const UID="John"
Const PWD="1234"
'*******************************

4.4 SET UP SECURITY (OPTIONAL)

Constant
Explanation
Const HostName = ""

 

The constant Const HostName = "" needs to reflect the address of your website. (Ex. Const HostName = "adnow.com".)

 

 This constant is used to prevent clicks from the location other than your web site. For example, if someone saved your web page and the banners and later click on the banner (on the same day or other days), Clicksee Adnow won't record the click.

This function will detect where the origin of the clicks. If your web site is www.adnow.com, you should put only adnow.com into the field. This function will work.

Please be careful when using this feature. Make sure that your visitors are not viewing your web site in any other names such as viewing your web site using an IP address.
Const Count_check = ""

 

(Example, Const Count_check = "30")

 This security feature can protect you from someone that keeps reloading your single web page in a specific amount of time.

Both Constants Const Count_check = "" and Const Time_check = "" work together and therefore you MUST place a numeric value in the both fields. 

Here's how it works. 

Const Count_check = "30" and Const Time_check = "1" 

If a spammer keeps reloading your web page for more than 30 times in one minute, the banners still show up, but no impressions will be recorded on the 31st request onward.
Const Time_check = ""

 

(Example, Const Time_check = "1")

2. MANAGING THE ADVERTISEMENTS (FOR THE ADMINISTRATOR)

2.1 Set up your first advertisement

2.1.1 Add company
After logging in, you will see the Admin Center page, click "Add New Company." Fill out all the necessary information in the provided fields, then click Add Campaign.

NOTE: A campaign can contain one advertisement or more. It gives you a convenience of ad tracking. For example, you may sell one order that contains 1 banner and 1 button to an advertiser in January. Later when the campaign expire at the end of the month, you can setup a new campaign for February.

2.1.2 Add Campaign
Type in a Campaign Name and Campaign Description (Optional) then click Next to continue to add the advertisement.

2.1.3 Add New Advertisement
 Image Characteristics
Type in the name of the ad and then continue to add Image URL, Image width, Image height, Image border, Image ALT. These fields are optional if you want to set up a graphical link, they are not required for the text link ad (no graphic).

Link to URL
The "Link to URL" field should be identified with the entire URL that you want the ad to link to. Always put the http:// in this field.

The Link Target
With this option, you can specify how you want an ad's URL to open. (Ex. You want to open a new window when someone clicks on a specified link, just place "_blank" in this field. If your homepage contains frames, you can specify a frame name in this field too.)

The Location
Location refers to just the name, not a physical location, you give to the ad(s), and you will use this name to indicate where you want to put the ad(s). For example, you may have the banner on the top of your page and the button on the left of your page. You use "Location" to tell Adnow Version 2 that some ads will appear only at the "banner" location and some ads will appear only at the "button" location. You can name the location anything you want. The words "banner" and "button" are used as examples here only. This "Location" is also helpful when you want to sell the ad space on the front page and other pages separately. You may name the banner on the front page as "banner1" and the banner on another page as "banner2." The location name is not case-sensitive.

The Weight
The weight is the way to setup the frequency of the ads. Here's how it works. There are 10 levels of "weight." 10 is the highest and 1 is the lowest. If you have two ads and you give the weight of "2" to one ad and the weight of "1" to another ad, the ad that is assigned the weight of "2" will run for 2 times more than the ad that is assigned the weight of "1." For example, if the total impressions of the day is 1,500, the ad with the weight of "2" will be displayed for 1,000 impressions and the ad with the weight of "1" will be displayed for 500 impressions.

The Text Message Box.
Whatever you type into this box will appear under the image. You can place HTML tags in the text message box to customize the fonts. If you want to setup the text-only ad, please type in the text into this box and leave the Image URL field blank.
NOTE: If you are going to setup Rich Media Banners, Flash Banners, Java Applet Banners and HTML Banners, you must place the code in this text box.

Status
Please select your appropriate category:
1. Active
2. Expired
3. Hold
4. Hold to launch

Date Start
This is the actual start date of the campaign/ad.

Expiration Method:
You must select one of the three options available: Date End, Click Expire or # of Impressions Purchased.

Date End
With this option, you are allowing the advertiser to advertise on your website with an expiration date. (Ex. You sell the advertising space with an expiration date of 21 September 2020, when 21 September 2020 arrives, the ad will expire, regardless of the number of clicks or views)

Click Expire
With this option, the advertiser purchases a certain number of clicks on the ad, when a number of clicks on the advertisement has been reached, the ad will expire. (Ex. An advertiser purchases 10,000 clicks, when the advertisement has been clicked on for the 10,000th time, the advertisement expires.)

# of Impression Purchased
With this option, you are allowing the advertiser to basically purchase the number of times an ad appears on your website. (Ex. When 10,000 impressions are purchased on the index.asp page, when the index.asp page is viewed for the 10,000th time, the ad expires, regardless if the user clicks on the ad.)

After submit "Add New Company" and "Add New Advertisement", you can now start tracking your ad(s) through the TOOL: STATISTICS OF AVERAGE CLICK THRU BY LOCATION and TOOL: STATISTICS OF AVERAGE IMPRESSION BY LOCATION. You can Edit and Delete your advertisement and company information at will.

PART III - DISPLAYING ADVERTISEMENTS

We recommend you to view the file testad.asp in Redirect folder that is included in the zip for an example.

1. INSERT THE LINE(S) TO CALL THE INCLUDED FILE(S)

Your web pages have to be in .asp extension.

To display a banner on a particular page, simply insert this statement on the top of your page.

<!--#include file="adnow2/redirect/CSANV2_run.asp"-->

Make sure that it points to the correct directory that the file CSANV2_run.asp is in.

NOTE: If you set up the security (PART I, step 4.4), you will need to add this line on the top of your page.

<!--#include file="adnow2/redirect/check_click.asp"-->

 

2. INSERT THE LINE TO CALL AN AD

Insert the following line into your .asp web pages where you want the banner to appear.

<%adnow "Location_Name"%>

Replace Location_Name with the name of each location.

Banner Filtering

Clicksee Adnow V.2 also comes with the banner filtering feature. In some cases, you may want to have more than one banner in one web page. For example, you have one banner at the top and one banner at the bottom of the page. You don't want the same banner to appear in both top and bottom positions. You can use the banner filtering feature to accomplish this.

<%adnowv2 "Location_Name"%>

The Clicksee Adnow Version 2 provides the <%adnowv2 "Location_Name"%> which contains a filtering feature. You can insert a special code, <%adnowv2 "Location_Name"%> to instruct Clicksee Adnow to remember the first banner at the top and avoid displaying the same banner again at the bottom. 

Clicksee Adnow will try its best to not display the same ad. However, in some cases, the same ad may appear. For example, if there's only one banner in your web site, the software won't know what else to display and it will display the same banner on the top and bottom. If you have 2 banners on your web site and one banner is assigned the weight of 1 and the other banner is assigned the weight of 2, it's possible that the banner that has the weight of 2 will appear both on the top and the bottom. Tips: If you have a few banners on your web site, a weight of "1" should be set to all banners to avoid the repeating of banners caused by improper weight. If you have many banners (10 banners for example), the repetition is almost impossible to happen and you can assign the different levels of weights to each banner without worrying about the repeated banner. Clicksee Adnow will try its best to screen the repetition for you.

NOTE: Always use  <%adnow "Location_Name"%> unless you want filtering.

Appendix B
Setup ODBC

An instruction on how to setup an ODBC data source for Microsoft Access.

  1. Open ODBC Data Sources in Control Panel.
  2. Select System DSN tab, click Add.
  3. Select Microsoft Access Driver (*.mdb), click Finish.
  4. Type in the name in the Data Source Name field. You can name the data source anything you want, ClickseeAdnow2 for example.
  5. Under Database, click Select.
  6. Point to the directory that you create to store adnow.mdb in step 2.
  7. Under Database Name, highlight adnow.mdb, click OK.
  8. Click OK and OK again to exit.

An instruction on how to setup an ODBC data source for Microsoft SQL.

  1. Open ODBC Data Sources in Control Panel.
  2. Click on System DSN tab
  3. Click Add button, you will go to the ODBC Data Source Administrator page
  4. Select SQL Server, click Finish
  5. Enter the Data Source Name of your choice in Name field
  6. Enter Data Source description in Description field (Optional)
  7. Select the SQL server where your database resides, click Next
  8. Check with SQL Server authentication using a login ID and password entered by the user
  9. Enter SQL username in Login ID field, and SQL password in Password field, click Next
  10. Check change the default database to: option if the database name in the field below is not the one your database resided, then choose the database name from the dropdown list, click Next, click Finish
  11. Click Test Data Source button to test your SQL database setting
  12. You should receive TEST COMPLETED SUCCESSFULLY! message, if not please try to repeat the instruction again, click OK
  13. Click OK, then your will return to ODBC Data Source Administrator page again
  14. Click OK then you finished setting up ODBC for SQL Server
