Appendix A. Changes history
GetCellBorders


Returns the cell borders. 
mixed GetCellBorders ( integer $worksheet, integer $column, integer $row
         [, boolean $exclude_hidden = false [, boolean $exclude_merged = false ]] )
Parameters
$worksheet
Zero-based worksheet number (first sheet - 0, second - 1, and so on).
$column
Zero-based column index (first column (A) - 0, second (B) - 1, and so on).
$row
Zero-based row index (first row - 0, second - 1, and so on).
$exclude_hidden
See detailed explanation of how to use these parameters below.
$exclude_merged
See detailed explanation of how to use these parameters below.
Return Value

Returns the cell borders array.
Note: this function returns FALSE only when read_borders option is set to FALSE.

Remarks

exclude_hidden parameter

If $exclude_hidden is set to TRUE then cell borders will be retrieved with taking into account hidden coloumns and rows around the cell. If this parameter is set to FALSE or not specified at all then any hidden columns and rows around the cell will be considered as visible and cell borders styles will be returned using this information.
Here's some more detailed explanation that helps you to understand how to use this parameter.
Borders information are stored for each cell individually. Look at the Pic. 1 below:

Pic. 1

The red line can be defined as right-side line at B2 or as left-side at C2. At first sight there is no difference. Lets hide the B column. When the red line defined at B2 cell we'll got the result as shown at Pic. 2. If the red line at C2 cell the result will be as shown at Pic. 3.

Pic. 2 Pic. 3

$exclude_hidden parameter helps you to retrieve right border line style in these two situations. When this function called for A2 cell without specifying $exclude_hidden parameter or set in to FALSE it returns that no right border line defined for A2 cell regardless of visibility status of B2 column. When $exclude_hidden is set to TRUE, column B is hidden and this function called for A2 cell then ExcelExplorer will analyze the visibility status of B columns and returns no right line for Pic. 2 situation and red line defined at right side for Pic. 3.

exclude_merged parameter

Lets examine the following example:

Pic. 4

If you dont not specify $exclude_merged parameter of set it to FALSE then this function called for B2 cell returns the following borders information:
- left and top lines are solid;
- right and bottom lines are absent.
If you set $exclude_merged parameter to TRUE then ExcelExplorer returns the following info about borders in B2 cell:
- left and top lines are solid;
- right line is solid (as for the D2 cell);
- bottom line is solid (as for the B4 cell).
So with this parameter you'll got the border lines that defined for right and bottom lines not for A2 cell but for the whole merged area.
However in more complicated cases (Pic. 5) you need to call this function for each cell individually to get all border line styles.

Pic. 5

In this case function called without $exclude_merged parameter for D3 cell returns that only right-side thick line is defined for this cell.
This is useful when you need to display the whole merged cells area with one HTML table cell.
