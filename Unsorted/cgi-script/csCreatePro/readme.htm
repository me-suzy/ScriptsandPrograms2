Installation Instructions

We offer professional installation but many people like to install their own applications or are behind a firewall that does not allow external access. Follow the instructions below and you should not have any trouble. If you run into difficulty do not hesitate to ask for assistance.

Quick Install for Experienced Installers
1. Unzip the file and transfer the entire csCreatePro directory to a directory accessible from a web browser on your server. We suggest creating a directory called 'cgi-script' and then transferring the entire csCreatePro directory and contents from the zip file to this directory making sure all but the images are transfer in ASCII mode (images should be transferred in binary mode).

2. CHMOD csCreatePro.cgi to 777 and then open the URL with your web browser. If all goes well, this will give you the auto-setup screen. Change the username and password if you so desire and click save. You should then see the login screen where you can enter the control panel and start using the script. 

NOTES
1. If you keep coming back to the install screen, then chances are your web server doesn't have write access to your directories.. You will need to CHMOD the csCreatePro directory and image_upload subdirectory to 777.

2. If you are unable to run scripts outside your cgi-bin directory then you will need to create a separate directory in your root directory and upload the images, icons, and image_upload directory and contents there. Then, you will need to modify the htmlurl and htmlpath variables to point to this new location. Also in this directory, place the following files: csCreatePro.css, selcolor.html, and help.html.

Normal Installation Instructions
1. Unzip the file and transfer to your server with the following directory structure:

/cgi-script <-- cgiscript.net script directory
/cgi-script/csCreatePro <-- csCreatePro directory
/cgi-script/csCreatePro/images
/cgi-script/csCreatePro/icons
/cgi-script/csCreatePro/image_upload

csCreatePro.cgi should be located at
[root url]/cgi-script/csCreatePro/csCreatePro.cgi
where [root url] is the URL to your website. Example:
http://www.mycompany.com/cgi-script/csCreatePro/csCreatePro.cgi

2. CHMOD csCreatePro.cgi to executable (CHMOD 755). Ask your server administrator for assistance if required.

3. If your Perl executable is not located at /usr/bin/perl, edit the first line of csCreatePro.cgi to reflect the true location.

4. Run csCreatePro.cgi from your webrowser to initiate the auto-setup mode.

Example: http://www.mycompany.com/cgi-script/csCreatePro/csCreatePro.cgi

When the script is run for the first time, it goes into setup mode. A setup screen will appear where you can verify and modify the setup variables. Most of the time, there will be no need to modify any of these variables except the username and password. Those control access to the control panel and you can change then to anything you desire.

When you click the save button on the installation screen, it should automatically create a file called setup.cgi which will look something like:

$cgiurl = 'http://www.cgiscript.net/demo/csCreatePro/csCreate.cgi';
$cgipath = '/www/vhosts/cgiscript.net/demo';
$htmlurl = 'http://www.cgiscript.net/demo/csCreatePro';
$htmlpath = '/www/vhosts/cgiscript.net/demo/csCreatePro';
$rooturl = 'http://www.cgiscript.net/';
$rootpath = '/www/vhosts/cgiscript.net/';
$makebackups=1;
$username = 'demo';
$password = 'demo';
1;

Note: If you keep returning to the setup screen then your web server doesn't have the ability to write to your directories. If that is the case, a work-around is to CHMOD the csCreatePro and image_upload directory to 777. In addition, you'll need to CHMOD any file or directory to 777 that you want to manage with csCreatePro.

Hint: Deleting the setup.cgi file will trigger the script to go back into setup mode.

Installing the script in your cgi-bin directory

Due to the limitations of cgi-bin directories, you'll need to create 2 directories when installing the script in the cgi-bin directory. One directory inside cgi-bin that will contain the script and supporting files, and another directory outside the cgi-bin directory to contain the images. You will need to copy the images, image_upload, and icons directory to this newly created directory outside your cgi-bin directory and point the htmlurl and htmlpath variables in the setup.cgi file to this location. Also in this directory, place the following files: csCreatePro.css, selcolor.html, help.html and t_add_href.htm.

If you have any problems or questions with the installation, don't hesitate to email support@cgiscript.net. We also offer installation services.
Usage Instructions

Go to http://www.yourdomain.com/installdir/CSCreatePro.cgi?command=login where http://www.yourdomain.com/installdir/CSCreatePro.cgi is the URL to the csCreatePro on your server.

Importing Existing Pages into csCreatePro
1. Click the 'Manage Files' button.
2. Browse to the file you want to import.
3. Select the option in the dropdown for 'Import a Page'

Creating a New Page
1. Click the 'Manage Files' button.
2. Browse to the directory where you want to create the new file.
3. Select the option in the dropdown for 'Create a Page'

Managing your WebPages
Once a page is in the csCreatePro database, you can edit, remove it, delete it, and give users the ability to edit it. When the page is in the list, you will see the appropriate links next to it for editing. Removing a file means to just remove it from csCreatePro. Deleting a file physically deletes the file from your server. Only use the delete function if you truly want to delete this file off your server.

User Management
You can create new users or delete a selected user from csCreatePro with the appropriate buttons at the top of the management page. When you select the user in the dropdown box, it will show you a list of files they currently have access to edit. Additionally, their information will appear in the text fields where you can change any of the information (name, password, etc) and click the save changes button.

Assigning Pages to Users
Select the 'ALL' option in the user dropdown. You will see a list of all the files that are in the csCreatePro system. Find the file you want to give access to a particular user and click the 'Show Users' button. This will popup a window where you can add and remove users for that particular file.

Once a user has been created, they can log into csCreatePro with  http://www.yourdomain.com/installdir/CSCreatePro.cgi?command=login where http://www.yourdomain.com/installdir/CSCreatePro.cgi is the URL to the csCreatePro on your server.
