You can use in this template the following strings:

\$out{'error'} - error message if there is some problem
\$out{'title'} - page title (extracted from <title> tags)
\$out{'url'} - page url as entered by user
\$out{'description'} - page description (extracted from metatags)
\$out{'keywords'} - keywords (extracted from metatags)
\$out{'email'} - email address (if published on page)
\$out{'company'} - the domain name with upper-cased first letter

NOTES: These strings will be replaced with real values generated by the script. All other \$ and \@ signs in your template must be replaced with \\\$ and \\\@. Edit the ACTION value of this form to make it pointing to the right script location.

If you have many modules and it is not possible to submit to all with one script call, you can split your modules collection in multiple directories and tell the script which is the directory where are the modules to use. For this purpose pass to nph-submit.pl a variable dir with value the modules directory name. Make it as a hidden or as a pop-up select list and put it in your form. Default value is modules. For example if you add this line: <INPUT TYPE="HIDDEN" NAME="dir" VALUE="modules1"> the script will look for modules in modules1 directory.
