WebCards Upgrade Guide - Upgrade to version 1.0 final

Introduction

Please note: If you are upgrading from beta 4 to version 1 final you do not need to run an upgrade script.
Simply upload all of the files inside the /upload/ folder excluding the config.php file.

The upgrade files are located inside this (./upgrade) directory but you must ensure you use the correct one. They are not uploaded by default to your server as a security precaution.
Using the wrong upgrade script is likely to damage your database, rendering your WebCards installation unusable so always ensure you use the correct one.

This release sees the "gold" release of WebCards version 1. It includes 3 upgrade scripts to upgrade sequentially from previous versions. Read the note at the top of the page if you are upgrading from beta 4.

If you wish to upgrade you must do so sequentially. If you currently have beta 1 installed and would like to upgrade to this version you must upgrade to beta 2, then upgrade that to beta 3 and finally upgrade to beta 4. This ensures that all changes are made to your installation.

Upgrade process

Upgrading is a very simple process.
  • First of all you should backup your database in case anything goes wrong. You can do this with a program such as PHPMyAdmin. Most hosts supply such software but it can be downloaded free from the website.
  • All of the files used in WebCards have been heavily modified. You will not be able to reuse any of your old templates or language files.
  • IMPORTANT: you should not overwrite your config.php file or else you will lose all of your settings.
  • Now you should run the upgrade script upgrade_beta3_to_beta4.php to upgrade from beta 3 to beta 4.
  • You should then upload all of the files that came with the new version except config.php. This file contains important information that should not be overwritten during the upgrade process.
  • Finally, you should log into the administration centre and check the configuration section to ensure all the information is still correct. You should also send a test WebCard to ensure the process went smoothly.
  • VERY IMPORTANT: you should remove the upgrade_beta3_to_beta4.php file from your server as it can damage your system if somebody accesses it after you have upgraded.
If you are having trouble upgrading you can visit the WebCards Website for support.
