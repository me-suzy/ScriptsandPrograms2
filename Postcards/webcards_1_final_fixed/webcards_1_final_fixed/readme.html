WebCards Installation Guide

Thank you for downloading and installing WebCards, an e-card script that will keep visitors returning to your site time and time again. This is the first stable WebCards release, version 1.0. This version is stable and fully supported.

This guide will aid you in installing WebCards onto your system.

This guide is divided into 3 chapters:
  • Requirements
  • Upgrading
  • Installation
Requirements

The following are all required to be installed on your web server before installation can take place:
  • PHP version 4.0.2 or greater is required
  • MySQL version 3.22 or higher is required.
  • A method of sending mail. WebCards supports the standard PHP mail() function as well as SMTP. You must be able to send mail to use WebCards.
  • Please note: Webcards will work in a restricted way on systems with safe mode switched on
  • Please note: WebCards does not require its own database. It can share a database happily with any other applications you have
If you have met all the requirements please proceed to Installation or Upgrading.

Upgrading

No database changes have been made since the final beta version. You should upload all of the files in the /upload/ directory except config.php. Overwriting this file will result in you losing your configuration data.

Installation

Please follow the steps below to install Webcards:

Uploading
  1. Choose a directory where you want to install Webcards. It is recommended that you install in a folder not more than 1 directory deep, however it will not affect your installtion if you do so. A choice would be /webcards
  2. Upload the contents of the upload directory on your local system to the folder you created in the last step. All files must stay in their original directories. If your FTP client will not allow you to upload folders, create them manually and then upload the contents.
  3. Once the directories and files are in place, you will need to chmod some files and folders to allow the script to edit them. Follow the chmod guide below:
    • /export - chmod 0775
    • /images - chmod 0775
    • /images/thumbs - chmod 0775
    • /lang - chmod 0775
    • /templates - chmod 0775
    • /config.php - chmod 0666
    • /templates/webcards.css - chmod 0666
    • /templates/template.html - chmod 0666
    • You may also need to chmod the directory WebCards is installed in to 0775
    If you get permission errors when you run the program, try changing the CHMOD values to 0777. This may be necessary if you are running a Windows server.
Running the Installation Program
  1. You are now ready to use the installer. Point your web browser to http://www.domain.com/webcards/install.php or wherever you have your WebCards installation set up.
  2. The script will perform an integrity check to ensure all the necessary files and folders are present. If any errors are you found you will be instructed on how to fix them. If no errors are found just click on "Start Installation" to begin.
  3. The script will now check the files you have uploaded to ensure all the correct files are present and have sufficient permissions for the script to write to them.
  4. The next stage asks you to confirm the installation path and URL. The path (or base directory) is the absolute path to the WebCards installation. It must have a trailing slash. In most cases the default value is correct.
    The base URL should contain a trailing slash and is the URL to your WebCards installation. The default value is usually correct.
  5. The next step asks for your database information. Choose your database driver (MySQL). Then enter the host your database is on. "localhost" (without the quotes) is usually correct.
    The database table prefix allows you to add a prefix to all the tables in your WebCards installation to prevent conflicts with other tables. This is highly recommended if there are already tables present in your database.
    You must know your database username and password. Enter them next in the appropriate boxes.
    Enter the name of the databse you wish to install into. This databse must already exist.
  6. The final step is where you set up your Admin Centre username and password. The email address you enter will be the "reply-to" address attached to all notification emails and can be changed at any time.
  7. Finally, click "continue" and the script will check all the information you entered.
  8. If you are presented with any errors, go back and correct them. If all is OK, click the link at the bottom of the page to go to the Administration Centre.

    9. If you are having trouble installing the program you can visit the WebCards Website for support.
