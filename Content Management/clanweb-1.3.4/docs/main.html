Setup

Requirements

To run ClanWeb content management system you are required to have following things;

  • A web server or a account at a web hosting provider.
  • PHP 4.0.5 or better (We don't know yet how CW works under PHP5. Running CW on a server with php5 is performed at own risk.)
  • MySQL 3.23.x or better.
  • A browser that supports javascript and cookies.

Additional requirements

  • Knowledge in HTML-coding and how to use a FTP application.

Installing ClanWeb

  • Unpack CW onto preferred location.
  • Upload all files to the web server without changing the structure.
  • Run install.php through your browser (Example: http://www.yourdomain.com/admin/install.php) Follow the instructions.
  • Copy the code from the textbox and paste it into a new text file. Save the file as cfg.php and upload to your admin directory.
  • Remove install.php and make a test run by logging in with your newly set username and password.

Prepare your site to be integrated with ClanWeb content management system.

This is going to get tricky, so read throught this section carefully and if you don't get it to work, read it again and make sure that you have done everything right. If it still fail to work then visit http://forum.clanadmintools.com and make a post.

In the folder you unpacked there is a folder called "code". In this folder you will find a set of files named "news", "news_front" and so on. These files contain the default code that you need to implant with your site. The easiest way to do this is to split your site into minor pages named news.php, main.php, games.php and so on. Each and every file will contain the code needed to collect the information from the database that is needed to present for example the clans members.

Don't link these files from the admin directory, that may result in that the files don't work correct or in a security breach. This is how it should be done.
news.html:
<html>
<head>
<title>Myclan newspage</title>
</head>
<body>
<h3>Latest news</h3>

<---- HERE I WANT MY NEWS --->

</body>
</html>

code/news_front.php:
<?php
/* This script is going to present all news written. */
require("../cfg.php"); // Make sure that the path to cfg.php is correct
@mysql_connect($dbhost, $dbuser, $dbpw);
@mysql_select_db ($dbname);

$sql = "SELECT * FROM " .$db_prefix. "news ORDER BY id DESC";
$query = mysql_query($sql);
while ($read=mysql_fetch_array($query))
{
$id=$read["id"];
$name=$read["name"];
$topic=$read["topic"];
$newspost=$read["newspost"];
$newstype=$read["newstype"];
$dates=$read["dates"];

echo"$topic - $dates <br/>\n";
echo"$name<br/>\n";
echo"$newspost\n";
echo"<br/><a href=\"news.php?id=$id\">Read more</a><br/>\n";
}
?>

Now paste the code into news.html and save it as frontnews.php. The file will now look like this.

Now upload this to your site root. Not the admin root. Run a test and see if it works. If it works as it should then repeat the procedure with the other files. If this shouldn't work then look for error messages, visit our forum and make a post(first look after simlar posts made by other users) and we'll help you.

Upgrade

Upgrading from 1.3.x to 1.3.4

Upgrading from older versions than 1.3.3 will force you to run a fullscale backup on your entire admin directory including your database. This is very important.

  • Upload the files from 1.3.4 and overwrite all old files including cfg.php.
  • Make a data-only dump on your database and drop the old database.
  • Make a fresh setup of ClanWeb 1.3.3 using the install_133.php file. Follow the instuctions.
  • Import the sql dump and make sure that all data gets saved without errors.
  • When everything else is done without errors then run 133_to_134.php and follow the instructions.

Use example_cfg.php and update it with your own data and upload into admin root directory.

Upgrading from 1.3.3 and 1.3.4prX

NOTE: Make a backup copy of your admin directory including your database. To make a database copy, use tools like PHPMyAdmin and make a dump.

  • Upload and overwrite all files in the directory. (Except cfg.php)
  • Run 133_to_134.php through your browser. Follow the instructions.
  • When the update is finished, remove 133_to_134.php

You're now running the new version of ClanWeb.

Bugs

Reporting bugs

To report a bug please use our forums. Just place [BUG] in the title and we'll look at it as fast as we can. But before you do, please check your server software and ,if you have any, modifications so that we can be sure that its the script not a third-party error. Please also be as detailed as you can, if possible list the steps required to duplicate the problem. If you have a fix which you are VERY SURE works and does not introduce further problems or incompatibilities please let us know.

Security bugs

If it's a security related bug then DON'T post in the forums, instead mail info@clanadmintools.com and describe how you did to expose this bug. This because we don't want evül people to take advantage of the bug before we can fix it.
