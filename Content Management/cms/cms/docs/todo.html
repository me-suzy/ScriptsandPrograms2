  • Undo / Redo logic to the SQL database interface, main algorithm descried here:
    • We have to have 3 log files will named as the following: sql.log  undo.log  redo.log
    • Steps to execute an add SQL statement:
      • Execute that SQL statement on our database.
      • If we get an error, print error message then exit this procedure.
      • Save that SQL statement in the sql.log file.
      • Get the id of the current inserted record.
      • Prepare a new SQL statement to delete that exact record, and save that statement in the undo.log file.
    • Steps to execute a delete SQL statement:
      • Save that SQL statement in the sql.log file.
      • Get all information / data of the record that we are going to delete (select command using same where section).
      • Prepare an add SQL statement could insert that record again in the database (including set the original id value).
      • Execute the original SQL statement (delete statement) on our database.
      • If we get an error, print error message then exit this procedure.
      • Save that SQL statement in the sql.log file.
      • Save that prepared add SQL statement in the undo.log file.
    • Steps to execute a update SQL statement:
      • Get the original data of the fields that we are going to update (select command using same where section).
      • Prepare a new update SQL statement that can return this record to its original case.
      • Execute the original update SQL statement on our database.
      • If we get an error, print error message then exit this procedure.
      • Save that SQL statement in the sql.log file.
      • Save that prepared update SQL statement in the undo.log file.
    • Steps to execute undo action (this function will not be active when undo.log file is empty):
      • Pick up the last inserted SQL statement to the undo.log file (remove it from that log file).
      • Execute this SQL statement on our database.
      • Pick up the last inserted SQL statement to the sql.log file (remove it from that log file also).
      • Save this SQL statement into redo.log file.
    • Steps to execute redo action:
      • Pick up the last inserted SQL statement to the redo.log file (remove it from that log file).
      • Execute that SQL statement in the normal way throw the logic for add, edit, and delete SQL statement that explained previously in this document.
    • Very important notification: this explained logic is really simple one but it is fair enough to be used in Khaled CMS system. But I have to refer here in this document to the limitation of like this logic and how you can come over it (if you really need that advanced feature in your database). We suppose that delete and update SQL statement will affect one record in the database (in our case it is page id). But in general case one delete or update command may affect more than one record, which meaning more than one SQL statement in the undo log file to undo the action taken. In like that cases you have to add an extra mechanism to grouping related SQL statements together (by adding same id number to the SQL statement line in the log file for an example).
