Change Log

Description V0.1:

  • Easy to use and administrate content management system.
  • Users can search in system content.
  • Functionality available for each page include:
    • Printable version.
    • Send that page URL to friend email.
    • Add to favorite list.
  • Powerful WYSIWYG editor for page content.
  • Dynamic site map.
  • Multi language system.
  • Easy to customize system (messages, header, footer).
  • Admin can add, edit, delete, activate, and deactivate any page in the system.
  • Crypt passwords in the database.
  • Admin tool for html META tags.
  • Support WAP technology, users can browse system content throw their mobile phones.

Description V0.2:

  • Functionality available for each page include PDF version now.
  • Two search mode are available now (normal where condition, full-text search "available in MySQL database only").

Description V0.3:

  • Valid W3C CSS and W3C HTML 4 design.
  • Easy to customize system by using css style sheet.
  • Admin can schedule any page in the system.
  • Fully web based install script.
  • Use ADOdb technology, so system is database portable.
  • Admin is able to control links order of the chilled pages.

Description V0.4:

  • Enhanced PDF conversion tool using dompdf open source project.
  • Add shopping cart system where admin can sell online any item browsed in one of website pages (using CashU cards).
  • Enhance install script by adding upgrade functions for both versions v0.2 & v0.3
  • Enhance WYSIWYG editor by wrapping content text around an image using the CSS Float attribute.
  • Database backup system, where you can export/import XML files.
  • Add, edit, delete multi language pages by system administrator.
  • User could switch between multi language pages by click on related icon.
  • Fix search bugs (search in Arabic characters, using AND logic).
  • Add full system documentation.
  • Give the system its new name: Rama

Description V0.41:

  • Sending gz-encoded data to web browsers that support compressed web pages.
  • Grant access to the admin tool based upon the clients IP address.
  • Upgrade to the latest dompdf version to be 0.31
  • Fix bug in the printable/pdf language page.
  • Convert files from PHP to HTML extensions, on an Apache Web Server using RewriteEngine in .htaccess
  • Warning in the index page will refresh into page-1.html after 10 sec.
  • Set global variable off for more secure environment.
  • Improve backup system, now exported XML file will saved directly on your local machine (no need to copy and past any more).
  • SQL statements are now fully portable (use ADOdb limit function, and also use ADOdb sequence techniques for auto-increment fields).
  • Upgrade system documentation.

Description V0.5:

  • Simple template system embedding PHP into the original template file (template.html).
  • Harden scripts security by using magic_quotes_gpc, SSL, htmlspecialchars, etc…
  • Fix some bugs in the page edit script.
  • Shift system images to /cmsimages directory and leave /images directory to be used by template.
  • Show last updated date, and last user edited the content.
  • Fix created date issue in the installation script.

Description V0.6:

  • Split "lang.php" file to be one file for each language (in language sub directory), which is easier to maintain.
  • Interactive module has been added to the page system. There are three interaction modes available in addition to normal case (with no interaction ability), these modes are:
    • User posts will be viewed by admin only (contact us case).
    • User posts will be available to all visitors directly (not moderated mode).
    • User posts needs admin confirmation before it be shown to all visitors (moderated mode).
    Administrator is able to delete any interactive entry in addition to confirm show or hide any posts in the moderated mode.
  • Fix the bug in the WML converting script.
  • Fix the bug in order links function.
  • Harden scripts security by coding some XSS-cleaner.
  • Review cart system. Administrator is able now to set in a config file a list of countries (s)he accept to shipping his/her good's to.

Description V0.61:

  • Add XSS-cleaner to the interactive input form.
  • Use transaction technology when needed (BeginTrans, CommitTrans, RollbackTrans).
  • Backup function includes now all database tables not only pages table records. Both download and upload backup files are now in zip format.  
  • ADOdb upgraded to v 4.61
  • DomPDF upgraded to v 0.3.3

Description V0.62:

  • Integrate PayPal payment processing with system shopping cart.
  • Add emailed form to the interactive module options.
  • Users now are able to browse thier own invoices when ever they want.
  • Default show/hide interactive add form related now to the interactive mode.
  • Admin can define now in config file the default status of the new added pages.
  • Add marquee bar to the system modirated by administrator.
  • Admin can define now in config file if he/she want to view page visits counter to all users or restrect that information to himself.

Description V0.63:

  • There are now separate template file for each language.
  • Administrator is able now to define required fields in the mailing form for each page.
  • System does database table optimization each time administrator makes a backup.
  • Harden system security against XSS attacks where we reject any change of IP address during session.
  • Develop RSS gateway for the website content.
  • More advance search system for Arabic language using lex's rules.
  • System will replace @ character by JavaScript code to stop harvisting email addresses robots.
  • Change the WYSIWYG javascript editor and use the cross-browser rich-text editor (RTE) which based on the designMode() functionality. This editor working fine with IE5+/Mozilla 1.3+/Mozilla Firebird/Firefox 0.6.1+/Netscape 7.1+ (old one was working on IE 5+ browsers only)
  • DomPDF upgraded to v 0.4.1
  • Fix broken links refer to ew.js in several scripts and css file in printable page.
