Upgrading

A guide to upgrading from a previous version. Read all that apply, eg, if you are upgrading from RC1, read the notes on RC2, RC3, etc all the way up to the latest version so you don't miss anything. You do not however have to run all the upgrade scripts, this is done automatically when you run the first one. Just check the other sections for template updates, config changes, etc.

From Release Candidate 1
A lot has changed since release candidate 1 so just delete everything from the server (files have been moved, deleted, added there is loads) and upload everything from the new package. Watch out not to overwrite your old config file though!

Also, if you are using .htaccess to create static URLs then you need to update your .htaccess. The only chance you need to make is add QSA into the [L], ie, change it to [QSA,L] or just copy it accross from the readme file or sample .htaccess file in the resources directory.

Run install/upgrade/from_rc1.php. This will take care of all the SQL changes. Once this is done remove the installation directory then go to the config editor as there are new config options. The date format you can leave as default as can you for default skin. Just check the setting for virtual path is correct.

From Release Candidate 3
Get the changed files and upload them. Now run install/upgrade/from_rc3.php. This will make the SQL changes and attempt to patch the templates that need changing. If you have some heavily customised templates though you may want to check <a href="{ROOT}admin.php">Admin</a> has been replaced by {ADMIN_LINK} (normally in overall_header).

It is recommended you take a look at the admin panel also as there are several changes to the config settings which you may want to change to suite your site. Also as there are JavaScript changes, do a hard refresh or empty your cache the first time you moderate a topic.

From version 1.0.0
Backup your files and data then grab the changed files and upload them over the existing files. Run install/upgrade/from_1.0.0.php and this will take care of the database changes. Delete the installation directory.

Add the follwing to your CSS for each of your skinsets.

Make the following template changes. In overall_footer find:

All rights reserved.

Replace with:

{ALLRIGHTSRESERVED}.

Find:

Powered by

Replace with:

{POWERED_BY}

In overall_header find:

value="Search!"

Replace with:

value="{SEARCH}!"

And you're done!

From version 1.1.0
Backup your files and data then grab the changed files and upload them over the existing files. Run install/upgrade/from_1.1.0.php and this will take care of the database changes. Delete the installation directory.

There is some new code that needs to be added to the CSS of each skinset you have (if you haven't created any, you just need to add this to Particle Blue in the skin editor).

If you have customised the templates you need to make the following change (if not see below). Open overall_header and find:

<html>

Replace with:

<html lang="{LANGUAGE_CODE}">

If you haven't customised the templates then go into the skin editor and using the drop down where it says Edit Templates, delete both overall_header and overall_footer. These will then revert to the base templates and you won't need to update them again unless you customise them!
